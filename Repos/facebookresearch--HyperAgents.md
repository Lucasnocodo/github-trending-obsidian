## 專案簡介

HyperAgents 是一個自我參照的自我改進代理系統，能夠優化任何可計算的任務。其核心機制是透過 Python 實現的 meta-agent 和 task-agent，使用者可以透過 `python generate_loop.py --domains` 指令來啟動代理，並將結果輸出到 `outputs/` 目錄。這個設計使得代理能夠在多個領域中進行自我優化，並且能夠根據需求動態調整策略。技術上，HyperAgents 使用了多個依賴庫，如 `gym` 和 `torch`，這些都是強大的工具，能夠支援強化學習和深度學習的需求。這樣的選擇使得系統能夠處理複雜的計算任務，並且具備良好的擴展性。

與其他類似工具相比，如 `0xGF/boneyard` 和 `NVIDIA/NemoClaw`，HyperAgents 的特點在於其自我改進的能力，這使得它在面對不斷變化的環境時能夠快速適應。實際使用中，這個系統能夠處理大量資料，並且在運行過程中可能會遇到安全性問題，因為它執行的是未經信任的模型生成代碼。這意味著使用者需要對潛在的風險保持警覺。這個專案目前處於活躍開發階段，並且有著良好的社群互動，但也面臨著一些未解決的問題，如實驗日誌的存取問題。總體來說，HyperAgents 是一個具有潛力的工具，適合需要在不斷變化的環境中進行自我優化的開發者和研究者。

**技術棧**：`Python 3.12` · `Docker` · `Gym` · `Torch`

## 重點功能

- 自我改進代理 — 能夠根據環境變化自我調整策略，優化計算任務。
- 多領域支持 — 支援多個計算領域的優化，透過 `--domains` 參數選擇。
- 實驗日誌管理 — 提供實驗日誌的多部分 ZIP 存檔，便於分析和回顧。
- 安全性考量 — 提供警告，提醒使用者注意執行未經信任代碼的風險。
- 可視化分析 — 內建分析腳本，能夠生成圖表以便於數據分析。

## 快速開始

1. 安裝依賴
```bash
sudo dnf install -y python3.12-devel graphviz graphviz-devel cmake ninja-build bzip2-devel zlib-devel ncurses-devel libffi-devel
```
2. 建立虛擬環境
```bash
python3.12 -m venv venv_nat && source venv_nat/bin/activate
```
3. 安裝 Python 依賴
```bash
pip install -r requirements.txt && pip install -r requirements_dev.txt
```
4. 建立 Docker 容器
```bash
docker build --network=host -t hyperagents .
```
5. 啟動初始代理
```bash
bash ./setup_initial.sh
```

## 架構分析

HyperAgents 採用 Python 和 Docker 的架構，這使得其能夠在多平台上運行。系統的資料流從使用者輸入 API 金鑰開始，然後透過虛擬環境安裝所需的依賴，接著建立 Docker 容器以便於隔離環境。核心的 meta-agent 和 task-agent 透過 `generate_loop.py` 進行調度，並根據不同的計算任務進行自我優化。選擇 Docker 使得部署過程簡單，但也帶來了額外的學習成本和環境配置的複雜性。擴展性方面，這個系統能夠處理大量資料，但在面對極端情況時，可能會遇到性能瓶頸，特別是在計算資源有限的情況下。
