## 專案簡介

HyperAgents 是一個自我參照的自我改進代理系統，旨在優化任何可計算的任務。這個系統的核心機制是透過生成 loop 來運行代理，並使用多個基礎模型進行任務執行。用戶可以通過執行 `python generate_loop.py --domains` 指令來啟動代理，並將輸出結果保存在 `outputs/` 目錄中。這種設計的優勢在於它能夠自我調整和改進，適應不同的計算任務，從而提高效率和性能。技術上，HyperAgents 使用了 Python 及 Docker 進行環境管理，並依賴多個庫如 `requests`、`pandas` 和 `gym` 來支持其功能。

這些選擇使得系統能夠輕鬆集成和擴展，並且能夠在多種環境中運行。與其他工具相比，如 `0x0funky/agent-sprite-forge` 和 `2aronS/Duel-Agents`，HyperAgents 的獨特之處在於其自我改進的能力，這使得它在處理複雜任務時更具靈活性和適應性。實際使用中，這個系統能夠處理大量數據，並且在性能上有良好的表現，但需要注意的是，執行未經信任的模型生成代碼可能會帶來安全風險。對於希望探索自我改進代理的研究者和開發者來說，這個專案提供了一個強大的平台。總體而言，HyperAgents 是一個處於 beta 階段的專案，適合小型團隊或個人開發者進行實驗和研究，但在生產環境中使用時需謹慎考量其安全性和穩定性。

**技術棧**：`Python 3.12` · `Docker` · `requests 2.32.4` · `pandas 2.3.2` · `gym 0.23.0`

## 重點功能

- 自我改進代理 — 能夠根據任務需求自我調整和優化。
- 多領域支持 — 支援多種計算任務，透過 `--domains` 參數選擇。
- Docker 環境管理 — 使用 Docker 容器簡化安裝和部署。
- 詳細的實驗日誌 — 提供實驗日誌下載，便於分析和回顧。

## 快速開始

1. 安裝必要的依賴
```bash
sudo dnf install -y python3.12-devel graphviz graphviz-devel cmake ninja-build bzip2-devel zlib-devel ncurses-devel libffi-devel
```
2. 建立虛擬環境並安裝需求
```bash
python3.12 -m venv venv_nat && source venv_nat/bin/activate && pip install -r requirements.txt
```
3. 建立 Docker 容器
```bash
docker build --network=host -t hyperagents .
```
4. 設定初始代理
```bash
bash ./setup_initial.sh
```
5. 運行代理
```bash
python generate_loop.py --domains
```

## 架構分析

HyperAgents 的架構基於 Python 和 Docker，設計目的是為了提供一個靈活的環境來運行自我改進的代理。系統的資料流從用戶輸入的任務開始，經過多個代理進行處理，最終輸出結果。選擇 Docker 作為環境管理工具，因為它能夠簡化依賴管理和部署過程，但代價是需要額外的學習成本。
