## 專案簡介

HyperAgents 是一個自我參照的自我改進代理架構，旨在優化各種可計算的任務。使用者需先設置 API 金鑰，然後透過 Docker 環境進行安裝與運行。核心流程包括創建虛擬環境、安裝依賴並運行 `generate_loop.py`，該腳本會根據指定的領域生成代理行為。這個設計的賣點在於其自我改進的能力，能夠在多次迭代中優化性能，具備高度的靈活性和擴展性。技術上，HyperAgents 使用 Python 作為主要開發語言，並依賴於多個庫如 `requests`、`pandas` 和 `gym`，這些都是在機器學習和數據處理中非常常見的工具。這使得它在處理複雜計算任務時具備良好的性能表現。

與其他類似工具相比，如 `0xGF/boneyard` 和 `AgentSeal/codeburn`，HyperAgents 的獨特之處在於其自我改進的能力，這意味著它能夠在運行過程中不斷學習和調整策略，而不僅僅是依賴預先定義的規則。這一特性在需要持續優化的場景中尤為重要，例如自動化測試或持續集成環境。使用者可能會面臨的挑戰包括如何有效地設置和管理 API 金鑰，以及如何處理模型生成的代碼可能帶來的安全風險。這個專案目前處於活躍開發階段，適合對自我改進算法和自動化任務優化有興趣的開發者。未來幾個月，隨著社群的反饋和功能的增強，預計會有更多的應用場景和改進方向。對於小型團隊或個人開發者來說，這是一個值得嘗試的工具，尤其是在需要自動化和優化的專案中。

**技術棧**：`Python 3.12` · `Docker` · `requests` · `pandas` · `gym`

## 重點功能

- 自我改進能力 — 代理能在多次迭代中優化性能，適應不同的計算任務。

## 快速開始

1. 設置 API 金鑰
```bash
OPENAI_API_KEY=... ANTHROPIC_API_KEY=... GEMINI_API_KEY=...
```
2. 安裝依賴
```bash
sudo dnf install -y python3.12-devel graphviz graphviz-devel cmake ninja-build bzip2-devel zlib-devel ncurses-devel libffi-devel
```
3. 創建虛擬環境並安裝庫
```bash
python3.12 -m venv venv_nat && source venv_nat/bin/activate && pip install -r requirements.txt
```
4. 構建 Docker 容器
```bash
docker build --network=host -t hyperagents .
```
5. 運行代理
```bash
python generate_loop.py --domains
```

## 架構分析

HyperAgents 採用模組化架構，將不同功能分散在多個子目錄中，如 `agent/`、`analysis/` 和 `domains/`。這樣的設計使得各個模組可以獨立開發和測試，降低了整體系統的耦合度。資料流方面，主要通過 `generate_loop.py` 進行控制，該腳本負責調用不同的代理進行任務執行。

選擇 Python 作為開發語言，因為其在數據科學和機器學習領域的廣泛應用，並且有豐富的庫可供使用。這樣的選擇使得開發者能夠快速上手並進行擴展。擴展性方面，雖然模組化設計提供了靈活性，但在處理大規模數據時，可能會遇到性能瓶頸，特別是在多個代理同時運行的情況下。

整體來看，這個架構適合快速迭代和實驗，但在長期運行和大規模應用時需要進一步優化。
