## 專案簡介

HyperAgents 是一個設計用來創建自我參照、自我改進的代理系統，能夠針對任何可計算的任務進行優化。使用者需要先設置 API 金鑰，然後透過 Docker 環境進行安裝，這樣可以確保依賴的環境一致性。安裝完成後，使用者可以透過 `python generate_loop.py --domains` 指令來啟動代理，並將輸出結果保存在 `outputs/` 目錄中。這個系統的核心在於它的 meta-agent 能夠自我調整和改進，這樣的設計使得它能夠在多次迭代中學習和優化性能。HyperAgents 使用了 Python 作為主要開發語言，並依賴於多個庫如 `requests`、`pandas` 和 `torch`，這些選擇使得其在數據處理和機器學習方面具備強大的能力。

與其他工具相比，HyperAgents 的自我改進能力是其主要賣點，這意味著它能夠在不斷變化的環境中持續優化，而不僅僅是執行預定義的任務。這種設計使得它在需要長期運行和持續改進的應用場景中，特別適合使用。使用者需要注意的是，這個系統涉及執行未經信任的模型生成代碼，存在一定的安全風險，因此在使用時需謹慎。整體來看，這是一個仍在開發中的專案，適合對自我改進代理有興趣的研究者和開發者。

**技術棧**：`Python 3.12` · `Docker` · `requests 2.32.4` · `pandas 2.3.2` · `torch 2.1.0`

## 重點功能

- 自我改進能力 — 能夠在多次迭代中學習和優化性能，無需手動調整。
- 多域支持 — 透過 `--domains` 參數，能夠在不同的任務領域中運行。
- Docker 環境 — 提供一致的運行環境，簡化依賴管理。
- 安全考量 — 提供警告以提醒使用者執行未經信任的代碼的風險。
- 實驗日誌 — 提供實驗結果的下載鏈接，方便使用者進行分析。

## 快速開始

1. 安裝必要的系統依賴
```bash
sudo dnf install -y python3.12-devel graphviz graphviz-devel cmake ninja-build bzip2-devel zlib-devel ncurses-devel libffi-devel
```
2. 創建虛擬環境並安裝依賴
```bash
python3.12 -m venv venv_nat && source venv_nat/bin/activate && pip install -r requirements.txt
```
3. 構建 Docker 容器
```bash
docker build --network=host -t hyperagents .
```
4. 設置初始代理
```bash
bash ./setup_initial.sh
```
5. 運行 HyperAgents
```bash
python generate_loop.py --domains
```

## 架構分析

HyperAgents 的架構設計為一個模組化的系統，主要由多個代理組成，包括 meta-agent 和 task-agent。這種設計使得不同的代理可以專注於不同的任務，並且能夠通過自我學習來不斷優化性能。資料流方面，使用者的輸入經過 `generate_loop.py` 進入 meta-agent，然後由 task-agent 處理具體任務，最終結果被保存到 `outputs/` 目錄。這樣的設計使得系統能夠靈活應對不同的計算任務，但也可能導致在高負載下的性能瓶頸。整體來看，這種模組化的設計能夠提高系統的可擴展性，但在實現上需要考慮到各個模組之間的協調與通信。

## 技術深入分析

HyperAgents 的核心技術機制在於其自我參照的設計，允許代理在執行任務的同時進行自我優化。這一點通過 meta-agent 和 task-agent 的協作實現，meta-agent 負責高層次的決策，而 task-agent 則專注於具體的任務執行。該系統能夠處理多達數千個任務的數據，並且在性能上具備良好的擴展性，能夠在多個環境中運行。選擇 Python 作為開發語言，使得其在數據處理和機器學習方面具備強大的能力，但也可能導致在高負載下的性能瓶頸。這個專案的依賴樹相對較重，包含多個大型庫，如 `torch` 和 `pandas`，這可能會增加部署的複雜性。技術風險方面，執行未經信任的代碼可能導致安全問題，使用者需謹慎。整合方面，該系統能夠與主流的機器學習框架良好整合，但在 CI/CD pipeline 中的使用仍需進一步測試和優化。
