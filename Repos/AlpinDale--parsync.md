## 專案簡介

parsync 是一個高效的同步工具，專為 SSH 遠端和本地之間的文件傳輸設計，支持可恢復的傳輸和並行文件處理。用戶可以透過簡單的命令，如 `parsync -vrPlu user@example.com:/remote/path /local/destination`，實現快速的文件同步，並且支持自定義 SSH 端口。這個工具的設計重點在於高吞吐量，允許用戶在傳輸過程中斷開連接後能夠繼續進行，這在大型文件或不穩定網路環境中尤為重要。技術上，parsync 使用 Rust 語言開發，並依賴於如 `rayon` 進行並行處理，這使得它在性能上相較於傳統的 rsync 有顯著提升。與其他工具相比，如 `rsync` 和 `scp`，parsync 提供了更好的性能和可恢復性，特別是在需要頻繁中斷和重連的情況下。

這使得它在需要大量文件傳輸的場景中更具吸引力，尤其是對於 DevOps 和系統管理員。雖然目前支持的操作系統包括 Linux、macOS 和 Windows，但在 Windows 上的支持仍然是最佳努力，可能會有一些限制。使用者可能會發現，對於小型文件的傳輸，parsync 的優勢不如大文件明顯，因為其設計重點在於處理大量數據時的效率。整體來看，parsync 是一個值得考慮的工具，特別是對於需要高效文件同步的用戶。

**技術棧**：`Rust 2021` · `Cargo` · `Makefile`

## 重點功能

- 高吞吐量 — 支持並行文件傳輸，顯著提升傳輸速度。
- 可恢復傳輸 — 在網路中斷後可繼續未完成的傳輸。
- SSH 支持 — 透過 SSH 進行安全的文件傳輸，支持自定義端口。
- 多平台支持 — 同時支持 Linux、macOS 和 Windows 系統。
- 簡單易用 — 透過簡單的 CLI 命令進行操作，易於上手。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 同步文件
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用非預設 SSH 端口
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.
