## 專案簡介

parsync 是一款高效的同步工具，專為 SSH 遠端和本地到本地的檔案傳輸設計，支持恢復功能和並行檔案傳輸。使用者可以透過簡單的命令，例如 `parsync -vrPlu user@example.com:/remote/path /local/destination`，輕鬆地將遠端檔案同步到本地。這個工具的設計重點在於高吞吐量，能夠有效利用網路帶寬，並且在傳輸過程中支持區塊增量同步，這意味著即使在網路中斷後也能夠恢復傳輸，避免重複傳輸已經完成的部分。技術上，parsync 使用 Rust 語言開發，依賴於多個庫來實現 SSH 通訊、檔案處理和並行運算，這使得它在性能上具有優勢。

與傳統的 rsync 相比，parsync 的並行傳輸特性能顯著提高傳輸速度，特別是在處理大量小檔案時。這個工具的設計考量了跨平台的需求，支持 Linux、macOS 和 Windows 系統，並且提供了簡單的安裝方式，無論是透過 curl、PowerShell 還是 cargo 都能輕鬆安裝。使用者在實際操作中可能會遇到 SSH 配置問題，這需要確保 SSH 端口和用戶名正確配置。整體而言，parsync 是一個穩定且高效的選擇，適合需要高效檔案同步的開發者和系統管理員。

**技術棧**：`Rust 2021` · `Cargo` · `Makefile`

## 重點功能

- 高吞吐量 — 支持並行檔案傳輸，顯著提高傳輸速度。
- 恢復功能 — 在網路中斷後能夠恢復傳輸，避免重複傳輸已完成的部分。
- 跨平台支持 — 支援 Linux、macOS 和 Windows 系統。
- 簡單安裝 — 可透過 curl、PowerShell 或 cargo 安裝，方便快捷。
- SSH 支持 — 透過 SSH 進行安全的檔案傳輸，支持自定義端口。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 同步檔案
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用非預設 SSH 端口
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## 架構分析

parsync 採用 Rust 語言實現，這使得它在性能和安全性上都有優勢。其架構設計為模組化，使用了多個依賴庫來處理 SSH 通訊、檔案操作和並行運算。資料流中，使用 SSH2 協議進行安全的檔案傳輸，並利用 Rayon 庫實現並行處理，這樣可以在多個檔案之間同時進行傳輸，顯著提高效率。

選擇 Rust 而非其他語言（如 Python）是因為 Rust 提供了更好的性能和內存安全性。這也意味著在高負載的情況下，parsync 能夠保持穩定的性能。擴展性方面，parsync 在處理大量小檔案時表現優異，但在處理單一大檔案時可能會受到網路帶寬的限制。

## 技術深入分析

parsync 的核心技術機制在於其並行檔案傳輸的能力，這是通過 Rust 的 Rayon 庫實現的，能夠在多個檔案之間同時進行傳輸，從而提高整體的傳輸速度。效能方面，parsync 能夠在良好的網路環境下達到高吞吐量，特別是在處理大量小檔案時，能夠充分利用帶寬。設計上選擇 Rust 而非其他語言，這使得它在性能和安全性上都有優勢，並且依賴樹相對簡單，避免了過多的外部依賴。技術風險方面，隨著使用者數量的增加，可能會出現性能瓶頸，尤其是在處理極大檔案時。整合方面，parsync 可以輕鬆地與現有的 SSH 環境整合，並且在 CI/CD pipeline 中的使用也相對簡單，能夠快速上手。

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.

## Star 趨勢

> [!abstract]- Stars 成長追蹤
> ```dataviewjs
> const me = dv.page("Repos/AlpinDale--parsync");
> if (me?.star_history) {
>   const raw = me.star_history.toString();
>   const points = raw.split(",").map(p => { const [d, s] = p.split(":"); return { date: d, stars: parseInt(s) }; }).filter(p => !isNaN(p.stars));
>   if (points.length >= 2) {
>     const max = Math.max(...points.map(p => p.stars));
>     const lines = points.map(p => {
>       const w = Math.round(p.stars / max * 25);
>       return `${p.date} ${"\u2588".repeat(w)}${"\u2591".repeat(25-w)} ${p.stars.toLocaleString()}`;
>     });
>     const first = points[0].stars;
>     const last = points[points.length-1].stars;
>     const growth = first > 0 ? Math.round((last - first) / first * 100) : 0;
>     lines.push(`\n**成長** +${(last-first).toLocaleString()} stars（${growth}%）in ${points.length} snapshots`);
>     // 趨勢方向偵測
>     if (points.length >= 3) {
>       const mid = Math.floor(points.length / 2);
>       const fh = points.slice(0, mid), sh = points.slice(mid);
>       const rateF = fh.length > 1 ? (fh[fh.length-1].stars - fh[0].stars) / Math.max(1, (new Date(fh[fh.length-1].date) - new Date(fh[0].date)) / 86400000) : 0;
>       const rateS = sh.length > 1 ? (sh[sh.length-1].stars - sh[0].stars) / Math.max(1, (new Date(sh[sh.length-1].date) - new Date(sh[0].date)) / 86400000) : 0;
>       const ratio = rateF > 0 ? rateS / rateF : rateS > 0 ? 2 : 1;
>       const dir = ratio > 1.3 ? "Rising（加速中）" : ratio < 0.7 ? "Cooling（降溫中）" : "Stable（穩定）";
>       lines.push(`**趨勢方向** ${dir}（加速比 ${Math.round(ratio * 100) / 100}x）`);
>     }
>     dv.paragraph(lines.join("\n"));
>   } else { dv.paragraph("需要 2+ 次快照才能顯示趨勢"); }
> } else { dv.paragraph("尚無 star_history 資料（下次出現在 trending 時會開始追蹤）"); }

>   // 成熟度 (0-20)
>   const created = me.created ? new Date(me.created.toString()) : null;
>   const age = created ? Math.floor((Date.now() - created.getTime()) / 86400000) : 0;
>   const mat = age > 365 ? 20 : age > 180 ? 16 : age > 30 ? 10 : 5;
>   score += mat; breakdown.push(`成熟度: ${mat}/20`);
>   // 社群 (0-20)
>   const forks = me.forks || 0;
>   const comm = forks > 200 ? 20 : forks > 50 ? 15 : forks > 10 ? 10 : 5;
>   score += comm; breakdown.push(`社群: ${comm}/20`);
>   // 授權 (0-15)
>   const lic = me.license || "";
>   const friendly = ["MIT","Apache-2.0","BSD-2-Clause","BSD-3-Clause","ISC","Unlicense"].includes(lic);
>   const licScore = friendly ? 15 : lic && lic !== "N/A" ? 8 : 0;
>   score += licScore; breakdown.push(`授權: ${licScore}/15`);
>   const grade = score >= 80 ? "A" : score >= 60 ? "B" : score >= 40 ? "C" : "D";
>   const bar = "\u2588".repeat(Math.round(score/5)) + "\u2591".repeat(20 - Math.round(score/5));
