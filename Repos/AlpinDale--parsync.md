## 專案簡介

parsync 是一個高效能的同步工具，旨在透過 SSH 進行檔案的並行傳輸和可恢復的同步。使用者只需輸入 `parsync -vrPlu user@example.com:/remote/path /local/destination`，即可將遠端檔案同步到本地，並支援 SSH 配置的主機別名。這個工具的設計重點在於高吞吐量和可恢復性，特別適合需要大規模檔案傳輸的情境。技術上，parsync 使用 Rust 語言實作，並依賴多個庫如 `rayon` 進行並行處理，這使得它在處理大量檔案時能夠有效利用多核心 CPU 的優勢。與傳統的 rsync 相比，parsync 提供了更快的傳輸速度，並且在中斷後可以從上次傳輸的地方繼續，這對於不穩定的網路環境非常重要。

這個工具的依賴樹相對輕量，主要依賴於一些常見的 Rust 庫，這使得它的安裝和使用都相對簡單。相較於其他工具如 `rsync` 和 `scp`，parsync 在處理大檔案時的效率更高，並且支援 Windows 環境，這在類似工具中並不常見。使用者在實際操作中可能會遇到 SSH 設定的問題，特別是在使用非預設端口時，這需要額外的配置。整體而言，parsync 是一個穩定且高效的選擇，適合需要進行大規模檔案同步的開發者和系統管理員。

**技術棧**：`Rust 2021` · `Cargo` · `Makefile`

## 重點功能

- 高吞吐量 — 支援並行檔案傳輸，顯著提高同步速度。
- 可恢復性 — 在傳輸中斷後可從上次進度繼續，避免重複傳輸。
- 多平台支援 — 同時支援 Linux、macOS 和 Windows 環境。
- SSH 支援 — 透過 SSH 進行安全的檔案傳輸，支援自定義端口。
- 簡單安裝 — 可透過 curl 或 PowerShell 輕鬆安裝，或使用 Cargo 安裝。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 進行檔案同步
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用非預設 SSH 端口
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.
