## 專案簡介

parsync 是一個針對 SSH 遠端和本地文件同步的高效工具，支持並行文件傳輸和可選的區塊增量同步。用戶只需輸入 `parsync -vrPlu user@example.com:/remote/path /local/destination`，即可開始從遠端伺服器同步文件到本地，並且支持非預設的 SSH 端口。這種設計使得大文件的傳輸更加快速，並且在傳輸中斷的情況下可以輕鬆恢復，這是許多傳統 rsync 工具所無法做到的。技術上，parsync 使用 Rust 語言開發，並依賴於多個庫如 `rayon` 用於並行處理，`ssh2` 用於 SSH 通訊，這使得它在性能上有顯著的提升。與其他工具相比，如 `rsync` 和 `rclone`，parsync 的並行傳輸特性使得它在處理大量小文件或大文件時速度更快，特別是在高延遲的網絡環境中。

雖然 `rsync` 也支持增量同步，但其單線程的特性使得在大規模傳輸時效率較低。使用者在使用 parsync 時，可能會遇到 SSH 配置問題，特別是在使用非標準端口時，這需要額外的配置。這個專案目前處於 beta 階段，適合需要高效能文件同步的開發者和系統管理員，未來可能會引入更多的功能，如更細緻的錯誤處理和進度顯示。對於小型團隊或個人專案來說，這是一個值得考慮的選擇。

**技術棧**：`Rust 2021` · `Cargo` · `Makefile`

## 重點功能

- 高效能並行傳輸 — 支持同時傳輸多個文件，顯著提高傳輸速度。
- 斷點續傳 — 在傳輸中斷後可從中斷點繼續，避免重複傳輸。
- SSH 支持 — 使用 SSH 進行安全的文件傳輸，支持非預設端口。
- 簡單的命令行介面 — 使用者只需簡單的命令即可完成複雜的同步任務。
- 多平台支持 — 支援 Linux、macOS 和 Windows 系統，方便不同環境的使用。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 同步文件
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用非預設 SSH 端口
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## 架構分析

parsync 採用 Rust 語言開發，這使得其在性能和安全性上都有優勢。其核心架構基於並行處理，使用 `rayon` 庫來實現多線程文件傳輸，這樣可以充分利用多核處理器的性能。資料流方面，parsync 在傳輸過程中會先建立 SSH 連接，然後將文件分割成多個塊進行並行傳輸，這樣可以顯著減少傳輸時間。

選擇 Rust 而非其他語言如 Python，主要是因為 Rust 的性能優勢和內存安全性，這使得 parsync 在處理大文件時表現更佳。這種設計雖然增加了開發的複雜度，但最終帶來的性能提升是值得的。擴展性方面，parsync 可以輕鬆集成到現有的 CI/CD 流程中，但在某些情況下，對於特定的 SSH 配置可能需要額外的調整。

## 技術深入分析

parsync 的核心技術機制是基於 Rust 語言的並行處理能力，使用 `rayon` 庫來實現多線程的文件傳輸。這種設計使得在高延遲的網絡環境中，parsync 能夠顯著提高傳輸效率。具體來說，parsync 將文件分割成多個塊，並通過多個線程同時傳輸，這樣不僅加快了傳輸速度，還能在傳輸中斷時快速恢復。從效能上看，parsync 能夠處理大約數十 GB 的數據，具體取決於網絡帶寬和延遲。

選擇 Rust 而非其他語言如 Go，主要是因為 Rust 的性能和內存安全性，這使得 parsync 在處理大文件時表現更佳。依賴樹相對簡單，主要依賴於幾個核心庫，這降低了維護成本。技術風險方面，parsync 在高負載情況下可能會遇到性能瓶頸，尤其是在多用戶同時使用的情況下。對外部 API 的依賴程度較低，主要依賴於 SSH 連接，這使得其在安全性上有一定的優勢。

整合方面，parsync 可以輕鬆與主流 CI/CD 工具整合，但在某些情況下，對於特定的 SSH 配置可能需要額外的調整。

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.
