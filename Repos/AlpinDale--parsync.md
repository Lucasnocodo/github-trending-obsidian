## 專案簡介

parsync 是一個高效的同步工具，專為 SSH 遠端和本地之間的檔案傳輸設計，並支援可恢復的傳輸。使用者只需輸入 `parsync -vrPlu user@example.com:/remote/path /local/destination`，即可開始從遠端伺服器同步檔案至本地，並且支援非預設 SSH 埠的連接。這個工具的核心賣點在於其並行檔案傳輸的能力，透過使用 Rust 的 `rayon` 套件來實現多執行緒處理，顯著提升了傳輸速度。parsync 還支援區塊增量同步，這意味著即使在大檔案的情況下，僅需傳輸變更的部分，進一步減少了帶寬的使用。與傳統的 rsync 相比，parsync 在處理大量小檔案時表現更佳，因為它能夠同時處理多個檔案，降低了總的傳輸時間。

這個工具的設計考量了跨平台的需求，支援 Linux、macOS 和 Windows，並且在 Windows 上也提供了最佳化的支援。使用者在使用過程中，可能會遇到 SSH 配置的問題，特別是在使用自定義埠時，這需要在命令中明確指定。這個專案目前處於穩定的 beta 階段，適合需要高效檔案同步的開發團隊，尤其是那些在多平台環境中工作的團隊。未來的發展方向可能會集中在增強用戶界面和進一步優化性能上。

**技術棧**：`Rust 1.60` · `Makefile`

## 重點功能

- 高效並行傳輸 — 支援多檔案同時傳輸，顯著提升速度。
- 可恢復的同步 — 在傳輸中斷後可繼續，避免重複傳輸。
- 增量同步 — 僅傳輸變更的檔案區塊，節省帶寬。
- 跨平台支援 — 同時支援 Linux、macOS 和 Windows。
- SSH 支援 — 透過 SSH 安全地進行檔案傳輸。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 進行檔案同步
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用自定義 SSH 埠
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.
