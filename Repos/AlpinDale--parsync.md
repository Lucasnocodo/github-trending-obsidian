## 專案簡介

parsync 是一個高效的同步工具，專為 SSH 遠端和本地之間的檔案傳輸設計，支援恢復和並行傳輸。使用者只需輸入 `parsync -vrPlu user@example.com:/remote/path /local/destination`，即可開始從遠端伺服器同步檔案到本地，並且它支援非預設的 SSH 埠號。這個工具的賣點在於其高吞吐量和可恢復性，讓使用者在網路不穩定的情況下也能順利完成檔案傳輸。技術上，parsync 使用 Rust 語言開發，並依賴多個庫如 `rayon` 進行並行處理，這使得它在處理大量檔案時能保持高效能。相較於傳統的 rsync，parsync 提供了更快的傳輸速度和更好的錯誤恢復能力，特別適合需要頻繁同步大檔案的場景。

它的依賴樹相對輕量，主要依賴於 Rust 的標準庫和一些輔助庫，這使得安裝和使用都相對簡單。使用者在實際操作中可能會遇到 SSH 配置的問題，但這可以透過 SSH config 檔案輕鬆解決。這個專案目前處於 beta 階段，穩定性不錯，適合中小型團隊使用，未來可能會進一步增強對 Windows 的支援。對於需要在多平台上進行檔案同步的開發者來說，parsync 是一個值得考慮的選擇。

**技術棧**：`Rust 2021` · `Cargo` · `Makefile`

## 重點功能

- 高吞吐量 — 支援並行檔案傳輸，顯著提高傳輸效率。
- 可恢復性 — 支援在傳輸中斷後繼續同步，避免重複傳輸。
- SSH 支援 — 透過 SSH 進行安全的檔案傳輸，支援自定義埠號。
- 多平台支援 — 可在 Linux、macOS 和 Windows 上運行。
- 簡單安裝 — 提供多種安裝方式，包括 curl 和 cargo。

## 快速開始

1. 安裝 parsync
```bash
curl -fsSL https://alpindale.net/install.sh | bash
```
2. 使用 parsync 同步檔案
```bash
parsync -vrPlu user@example.com:/remote/path /local/destination
```
3. 使用非預設 SSH 埠號
```bash
parsync -vrPlu user@example.com:2222:/remote/path /local/destination
```

## README 摘錄

> [!info]- 展開查看原文 README
> # parsync
> 
> `parsync` is a high-throughput, resumable sync tool for SSH remotes and
> local-to-local transfers, with parallel file transfers and optional block-delta
> sync.
> 
> ## Installation
> 
> **Linux and macOS:**
> 
> ```bash
> curl -fsSL https://alpindale.net/install.sh | bash
> ```
> 
> **Windows:**
> 
> ```powershell
> powershell -ExecutionPolicy Bypass -c "irm https://alpindale.net/install.ps1 | iex"
> ```
> 
> You can also install with cargo:
> 
> ```bash
> cargo install parsync
> ```
> 
> You may also download the binary for your platform from the
> [releases page](https://github.com/AlpinDale/parsync/releases), or install from source:
> 
> ```bash
> make build
> make install
> ```
> 
> ## Platform support
> 
> - Linux: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`
> - macOS: `aarch64-apple-darwin`, `x86_64-apple-darwin`
> - Windows: `x86_64-pc-windows-msvc` (best-effort metadata support)
> 
> ## Usage
> 
> ```bash
> parsync -vrPlu user@example.com:/remote/path /local/destination
> ```
> 
> With non-default SSH port:
> 
> ```bash
> parsync -vrPlu user@example.com:2222:/remote/path /local/destination
> ```
> 
> SSH config host aliases are supported.
