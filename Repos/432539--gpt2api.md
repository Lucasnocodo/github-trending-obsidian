## 專案簡介

gpt2api 是一個針對 AIGC 服務的聚合網關，能夠將 GPT 和 GROK 的能力封裝成 OpenAI 兼容的 API。用戶可以透過簡單的 HTTP 請求來生成文字、圖片和視頻，並且支持批量處理和多帳號調度。這個系統的設計使得開發者能夠無縫接入 OpenAI SDK，並且在後端提供了完整的管理界面來控制帳號和代理。核心的 API 路由包括文字對話 (`POST /v1/chat/completions`)、文生圖 (`POST /v1/images/generations`) 和視頻生成 (`POST /v1/video/generations`)，這些功能都支持高並發請求。

技術上，後端使用 Go 和 Gin 框架，數據存儲則依賴 MySQL 和 Redis，這樣的選擇使得系統在性能上具備優勢。與其他類似工具相比，gpt2api 的多帳號池管理和代理策略更為靈活，特別適合需要高並發和穩定性的商業應用。這個專案的設計考慮到了用戶的需求，並提供了完整的文檔和快速部署指南，讓新手也能輕鬆上手。隨著使用者數量的增加，社群活躍度也在提升，這對於未來的功能擴展和問題解決都是有利的。

**技術棧**：`Go 1.24` · `Gin` · `GORM` · `MySQL` · `Redis` · `React 18` · `Vite` · `TypeScript` · `TailwindCSS` · `Docker` · `Docker Compose`

## 重點功能

- 多帳號池管理 — 支持批量導入和健康檢測，能夠自動刷新和熔斷。
- OpenAI 兼容 API — 提供完整的 OpenAI API 路由，支持文字、圖片和視頻生成。
- 代理池支持 — 能夠批量導入和測試代理，並支持固定和隨機代理策略。
- 統一計費系統 — 提供按模型、分辨率和時長的積分計費功能。
- 完整的管理後台 — 提供用戶管理、Token 管理和請求日誌的功能。

## 架構分析

gpt2api 採用微服務架構，後端使用 Go 和 Gin 框架，這使得系統能夠高效處理並發請求。用戶前台和管理後台分開部署，通過 Nginx 或 Caddy 進行反向代理和流量控制。數據存儲使用 MySQL 和 Redis，這樣的選擇能夠保證數據的快速讀取和寫入。

系統的設計考慮到了可擴展性，能夠支持未來的功能擴展和高並發請求。選擇 Go 作為後端語言是因為其高效的性能和良好的並發處理能力，這對於 AIGC 服務至關重要。整體架構的擴展性良好，但在高並發情況下可能需要進一步的資源配置來保持穩定性。

## 技術深入分析

gpt2api 的核心技術機制是將多個 AIGC 服務整合到一個統一的 API 中，這樣用戶只需通過一個接口即可訪問不同的生成模型。系統使用 Go 語言和 Gin 框架，這使得它在處理高並發請求時表現出色。數據存儲方面，MySQL 負責持久化數據，而 Redis 則用於快取和會話管理，這樣的設計能夠顯著提高系統的響應速度。設計上選擇微服務架構，這樣可以方便地擴展各個功能模塊，但也帶來了服務之間的通信和管理複雜性。技術上，這個專案依賴於 Docker 進行容器化部署，這樣能夠簡化環境配置和版本管理，但對於不熟悉 Docker 的開發者來說，可能會增加學習成本。整體而言，這個專案在性能和可擴展性上都有良好的表現，但在高負載情況下，仍需注意資源的合理配置。

> ## 项目简介
> 
> `gpt2api`（项目代号 KleinAI）是一个生产级的 **AIGC 聚合网关**，把 GPT / GROK 这类账号 + Cookie 体系的能力，整体封装成 **OpenAI 兼容协议**，让任何按 OpenAI SDK 编写的程序都可以无缝接入。
> 
> 平台同时提供：
> 
> - 一套面向终端用户的创作前台（图 / 文 / 视频）
> - 一套面向运营的管理后台（账号池、代理、计费、CDK、日志）
> - 一套对外暴露的 OpenAI 兼容 HTTP 接口
> 
> 适用场景：私有化 AIGC 服务、白标 SaaS、多账号合规分发、内部团队调用聚合。
> 
> > 当前默认版本：`v2.0.1`，建议直接以 `v2.0.x` 演进；`v1.0.x` 保留为历史稳定基线。
> 
> ## ✨ 功能特性
> 
> ### 创作能力
> 
> | 能力 | OpenAI 兼容路由 | 说明 |
> |------|----------------|------|
> | 文字对话 | `POST /v1/chat/completions` | 支持流式 / 非流式输出 |
> | 文生图 | `POST /v1/images/generations` | 支持批量出图、`gpt-image-2` 稳定通道 |
> | 图生图 | `POST /v1/images/edits` | 支持参考图、Mask |
> | 图片任务查询 | `GET /v1/images/generations/:task_id` | 异步任务进度 / 结果回查 |
> | 文 / 图生视频 | `POST /v1/video/generations` | 支持 `quality=standard\|hd`，对应 720p / 1080p |
> | 视频任务查询 | `GET /v1/video/generations/:task_id` | 异步任务进度 / 结果回查 |
> | 模型列表 | `GET /v1/models` | 由后端模型表统一暴露，可在管理后台维护 |
> 
> ### 调度与稳定性
> 
> - **多账号池**：GPT / GROK 账号批量导入、健康检测、自动刷新、熔断、轮换
> - **代理池**：批量导入（`scheme://user:pass@host:port#name`）、批量删除、批量测试
> - **代理策略**：账号级绑定优先，全局回落支持「固定代理」与「随机代理」两种模式
> - **请求观测**：上游全链路日志可追踪，失败任务可看到完整 provider 报文
> - **统一计费**：积分制，按模型 / 分辨率 / 时长可配置
> 
> ### 运营能力（管理后台）
> 
> - 仪表盘、Token（账号）管理、代理管理、用户管理、充值消费
> - 优惠码、CDK 兑换、模型价格、系统配置、请求日志、上游日志
> - 所有配置尽量表单化，避免裸 JSON 手填

> ## 🧩 OpenAI 兼容 API
> 
> 直接把 OpenAI SDK 的 `base_url` 指向本服务即可：
> 
> ```python
> from openai import OpenAI
> 
> client = OpenAI(
>     base_url="https://your-domain:17200/v1",
>     api_key="sk-xxxxxxxx",  # 在用户前台「密钥」页生成
> )
> 
> # 文字对话
> resp = client.chat.completions.create(
>     model="gpt-4o-mini",
>     messages=[{"role": "user", "content": "你好"}],
> )
> 
> # 图片生成
> img = client.images.generate(
>     model="gpt-image-2",
>     prompt="一只在京都樱花树下的赛博狐狸，电影质感",
>     size="1024x1024",
>     n=4,
> )
> 
> # 视频生成（v2.0.1 起默认 720p / 1080p）
> import httpx
> r = httpx.post(
>     "https://your-domain:17200/v1/video/generations",
>     headers={"Authorization": "Bearer sk-xxxxxxxx"},
>     json={
>         "model": "grok-video",
>         "prompt": "在雨夜霓虹中漫步的猫",
>         "aspect_ratio": "16:9",
>         "quality": "hd",      # standard=720p, hd=1080p
>         "duration": 6,
>     },
>     timeout=600,
> )
> print(r.json())
> ```

> ## ⚙️ 配置说明
> 
> ### 全局代理策略
> 
> 在「管理后台 → 系统配置」中：
> 
> - `proxy.global_enabled`：是否启用全局代理回落
> - `proxy.selection_mode`：`fixed`（固定代理） / `random`（随机代理）
>   - `fixed` 模式下需要在 `proxy.global_id` 指定一个代理
>   - `random` 模式下每次任务从启用代理中随机挑一个
> - 账号级绑定的 `proxy_id` 始终优先于全局策略
> 
> ### Token 账号管理
> 
> - 支持 GPT / GROK 双 Provider，导入时按行解析
> - 导入后会自动针对 GROK Cookie 账号触发探测，识别账号类型（`basic / super / heavy`）并回填到列表
> - 列表支持按「账号类型」过滤
> - 支持批量绑定代理：`single`（多账号绑同一代理） / `cycle`（多账号轮询绑定多个代理）
