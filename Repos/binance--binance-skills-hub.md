## 專案簡介

這個專案提供了一個開放的技能市場，讓 AI 代理人可以透過自然語言與加密貨幣互動。它的架構允許開發者為不同的 AI 框架（如 LangChain 和 CrewAI）創建技能，並且支持多種區塊鏈。與其他加密工具相比，這個專案的獨特之處在於它的通用性和易於擴展的設計，任何開發者都能輕鬆添加新技能。實際使用中，這個平台能夠快速執行交易和查詢，然而，使用者仍需對加密市場有基本了解以避免風險。整體來說，這是一個值得嘗試的專案，尤其適合希望將 AI 與加密貨幣結合的開發者。

**技術棧**：`Node.js` · `Express` · `MongoDB`

## 重點功能

- 支持多種加密貨幣交易和監控功能。
- 允許開發者為不同的 AI 框架創建技能。
- 提供自然語言介面，簡化與加密市場的互動。
- 每個技能都可以獨立管理和擴展。
- 設計為與整個加密生態系統兼容。

## 快速開始

1. 克隆專案
```bash
git clone https://github.com/binance/binance-skills-hub.git
```
2. 進入專案目錄
```bash
cd binance-skills-hub
```
3. 安裝依賴
```bash
npm install
```

## 為什麼值得關注

> [!tip] 爆紅原因
> Binance 作為知名的加密貨幣交易平台，其開發者背景和資源使得這個專案具備強大的支持。隨著 AI 技術的普及，市場對於能夠簡化加密交易的工具需求上升，這個專案正好切中這個需求。近期的加密市場波動也促使開發者尋找更高效的交易方式。

## 適合誰使用

**目標受眾**：對加密貨幣和 AI 技術有興趣的開發者和交易員。

> [!example] 使用場景
> - 交易員 用它來 自動執行交易，因為這樣可以節省時間並提高交易效率。
> - 開發者 用它來 創建自定義的交易策略，因為這樣能快速測試不同的市場假設。
> - 研究人員 用它來 監控市場信號，因為這樣可以即時獲取重要的市場動態。

## 架構分析

專案採用前後端分離架構，後端使用 Node.js 處理 API 請求，前端則提供用戶介面來與 AI 代理人互動。資料流是 用戶輸入 → API Server → AI 代理人推理 → 結果回傳前端。

## 優缺點分析

> [!success] 優點
> - 易於擴展，開發者可以快速添加新技能。
> - 支持多種加密貨幣和區塊鏈。
> - 自然語言介面使得使用者友好。

> [!danger] 缺點
> - 需要使用者對加密市場有基本了解。
> - 部分功能可能需要額外的 API 設定。
> - 仍在開發中，穩定性可能不足。

> [!warning] 注意事項
> - 需要對加密市場有基本了解。
> - 部分功能可能需要額外的 API 設定。
> - 目前仍在開發中，可能會有不穩定的情況。

> # Binance Skills Hub
> 
> Binance Skills Hub is an open skills marketplace that gives AI agents native access to crypto: both centralized and decentralized. Search tokens, execute trades, track wallets, monitor signals, and interact with DeFi protocols, all through natural language.
> 
> Built by Binance. Built for everyone.
> 
> We're not building this just for Binance products. Skills Hub is designed for the entire crypto ecosystem: any agent, any framework, any chain. Whether you're building on LangChain, CrewAI, or your own stack, your agents can plug into crypto with a few lines of config.
> 
> ---
> 
> ## About This Repository
> 
> Each skill lives in its own folder and contains a `SKILL.md` file with YAML frontmatter and structured instructions.
> 
> Browse the existing skills to understand patterns and naming conventions before contributing.
> 
> ---
> 
> ## Contribution
> 
> We welcome contributions.
> 
> To add a new skill:
> 
> 1. **Fork the repository** and create a new branch:
> 
>    ```bash
>    git checkout -b feature/
>    ```
> 
> 2. **Create a new folder** containing a `SKILL.md` file.
> 
> 3. **Follow the required format:**
> 
>    ```markdown
>    ---
>    title: 
>    description: A clear description of what the skill does and when to use it.
>    metadata:
>      version: 
>      author: 
>    license: MIT
>    ---
> 
>    # 
> 
>    [Add instructions, examples, and guidelines here]
>    ```
> 
> 4. **Open a Pull Request** to `main` for review.
>    Once approved, the skill will be merged.
>
