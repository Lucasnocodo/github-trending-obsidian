## 專案簡介

Codex Complexity Optimizer 是一個專為分析代碼庫設計的工具，能夠識別算法複雜度和性能瓶頸，並生成安全的優化報告。使用者只需在 Codex 中輸入 `$complexity-optimizer` 指令，即可獲得當前代碼的複雜度、建議變更、預期複雜度和風險等詳細資訊。這個工具的賣點在於它不會自動修改代碼，確保使用者在進行更改前能充分理解風險。若要應用建議的變更，則需明確要求進行實施，這樣可以有效避免潛在的錯誤。技術上，這個工具是基於 Node.js 開發，並使用 npm 進行安裝，這使得它能夠輕鬆集成到現有的開發環境中。

與其他工具相比，如 0xSero/codex-shim 和 0xGF/boneyard，Codex Complexity Optimizer 更加專注於複雜度分析，而不是僅僅提供代碼生成或簡單的代碼修復功能。這使得它在需要深入理解代碼性能的情境下更具優勢。實際使用中，這個工具能夠處理中小型代碼庫，並且不會對系統資源造成過大負擔，因為它的運行過程主要是分析而非實時編譯。雖然目前尚未有開放的問題，但隨著使用者增多，未來可能會出現一些使用上的挑戰。整體來看，這是一個適合希望提升代碼質量和性能的開發團隊使用的工具，特別是在代碼複雜度成為瓶頸的情況下。

**技術棧**：`Node.js`

## 重點功能

- 複雜度分析 — 提供代碼的當前複雜度和建議的變更。
- 安全報告 — 默認情況下不會自動修改代碼，確保使用者能夠理解風險。
- 風險評估 — 對每個建議的變更提供風險級別評估。
- 測試建議 — 提供必要的測試或基準建議以驗證變更效果。
- 簡單安裝 — 透過 npm 安裝，輕鬆集成到現有環境中。

## 快速開始

1. 全域安裝工具
```bash
npm install -g codex-complexity-optimizer
```
2. 在 Codex 中使用分析功能
```bash
Use $complexity-optimizer to analyze this codebase and give me a report.
```
3. 應用建議的變更
```bash
Use $complexity-optimizer to implement the lowest-risk optimization from the report and run the relevant tests.
```

> 
> ```bash
> npx codex-complexity-optimizer
> ```
>

> 
> In Codex:
> 
> ```text
> Use $complexity-optimizer to analyze this codebase and give me a report.
> ```
> 
> By default, report-only prompts do not modify files. The skill reports file/line, current complexity, recommended change, expected complexity after the change, risk level, and tests or benchmarks needed.

> 
> ```text
> Use $complexity-optimizer to implement the lowest-risk optimization from the report and run the relevant tests.
