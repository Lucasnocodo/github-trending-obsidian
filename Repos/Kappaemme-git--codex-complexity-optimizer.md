## 專案簡介

Codex Complexity Optimizer 是一個用於分析代碼庫的工具，能夠識別算法複雜度和性能瓶頸，並生成安全的優化報告。使用者只需在 Codex 中輸入指令 `$complexity-optimizer`，就能獲得包含當前複雜度、建議變更、預期變更後的複雜度、風險等級及所需測試的詳細報告。這樣的設計使得開發者能夠在不直接修改代碼的情況下，獲得優化建議，降低了代碼出錯的風險。若要應用建議的變更，則需明確請求執行最低風險的優化，這樣的流程確保了代碼的安全性和穩定性。技術上，這個工具是用 Python 和 JavaScript 開發的，並且依賴於 Codex 的生態系統，這意味著它能夠無縫整合到現有的開發流程中。

與其他類似工具相比，如 0xGF/boneyard 和 0xSero/codex-shim，Codex Complexity Optimizer 提供了更為安全的報告生成方式，並且不會自動修改代碼，這對於需要高穩定性的生產環境尤為重要。使用者在實際操作中可能會遇到的問題包括如何解讀報告中的風險評估，以及如何選擇合適的優化建議。這個專案目前處於初期階段，未來可能會增加更多的功能和優化建議。對於小型團隊或個人開發者來說，這是一個值得嘗試的工具，特別是當他們需要對代碼進行性能優化時。建議在非關鍵項目中先行試用，以評估其效果。

**技術棧**：`Python` · `JavaScript`

## 重點功能

- 代碼複雜度分析 — 提供當前複雜度和建議變更的詳細報告。
- 安全優化報告 — 預設不修改文件，降低代碼出錯風險。
- 風險評估 — 每個建議都附帶風險等級，幫助開發者做出明智的決策。
- 測試建議 — 報告中包含所需的測試或基準，確保變更的有效性。
- 易於安裝 — 只需一行命令 `npm install -g codex-complexity-optimizer` 即可安裝。

1. 安裝 Codex Complexity Optimizer

3. 應用建議的優化
```bash
Use $complexity-optimizer to implement the lowest-risk optimization from the report and run the relevant tests.
```

> 
> ```bash

> ```
>

> 
> In Codex:
> 
> ```text
> Use $complexity-optimizer to analyze this codebase and give me a report.
> ```
> 
> By default, report-only prompts do not modify files. The skill reports file/line, current complexity, recommended change, expected complexity after the change, risk level, and tests or benchmarks needed.

> 
> ```text
> Use $complexity-optimizer to implement the lowest-risk optimization from the report and run the relevant tests.
