## 專案簡介

Codex Complexity Optimizer 是一個專為分析代碼庫而設計的工具，能夠發現算法複雜度和性能瓶頸，並生成安全的優化報告。用戶只需在 Codex 中輸入指令 `$complexity-optimizer`，即可獲得當前代碼的複雜度、建議的變更、預期的複雜度以及相關的風險評估。這個工具的賣點在於其不會自動修改文件，使用者必須明確要求進行更改，這樣可以避免潛在的錯誤。技術上，它使用了 Node.js 進行包管理，並且安裝後會放置在用戶的 Codex 目錄中。這種設計使得它能夠輕鬆整合到現有的開發流程中，並且不會對代碼庫造成不必要的干擾。

與其他工具相比，如 0xGF/boneyard 和 0xSero/codex-shim，Codex Complexity Optimizer 更加專注於提供詳細的複雜度分析和風險評估，而不是僅僅進行代碼的自動優化。這使得它特別適合需要精細控制代碼質量的團隊。使用時，雖然它的報告非常詳細，但仍需使用者具備一定的代碼理解能力來評估建議的變更。整體而言，這個工具適合中小型團隊，特別是那些對代碼質量有高要求的開發者。

**技術棧**：`Node.js`

## 重點功能

- 複雜度分析 — 提供代碼的當前複雜度和建議的變更。
- 安全報告 — 預設不修改文件，避免潛在錯誤。
- 風險評估 — 每個建議都附帶風險級別，幫助使用者做出明智決策。
- 測試建議 — 提供實施變更所需的測試或基準建議。
- 簡單安裝 — 使用 npm 安裝，輕鬆整合到現有開發環境。
