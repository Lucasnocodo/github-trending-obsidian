## 專案簡介

claude-obsidian 是一個結合 Claude AI 和 Obsidian 的知識管理工具，旨在自動化筆記的整理與維護。使用者可以透過 `/wiki` 指令啟動工具，Claude 會讀取資料來源，生成 8-15 個維基頁面，並更新索引和日誌。這樣的設計讓知識隨著每次資料的加入而增長，類似於利息的複利效果。使用者只需將來源文件丟入，Claude 便會自動提取實體和概念，並更新交叉引用，確保知識庫的結構化。這個工具的賣點在於它不僅僅是回答問題，而是主動創建和維護知識，讓使用者能夠隨時獲得最新的上下文信息。

核心功能包括自動組織筆記、標記矛盾、會話記憶等，這些功能在其他 Obsidian 插件中並不常見。使用者可以選擇將其作為完整的 vault 克隆，或作為 Claude Code 插件安裝，並透過簡單的指令快速上手。這個工具的架構基於 Karpathy 的 LLM Wiki 模式，並支援多模型運作，讓使用者能夠靈活選擇使用的 AI 模型。相較於其他同類工具，claude-obsidian 提供了更強大的自動化和維護功能，特別適合需要長期維護知識庫的使用者。

**技術棧**：`Shell`

## 重點功能

- 自動組織筆記 — Claude 會自動創建實體和概念，並更新交叉引用。
- 會話記憶 — 熱緩存功能讓使用者在不同會話中保持上下文。
- 批量資料處理 — 支援同時處理多個來源，並進行交叉引用。
- 健康檢查 — 自動檢查維基的健康狀態，找出孤立的頁面和死鏈接。
- 多模型支援 — 支援 Claude、Gemini、Codex 等多種 AI 模型的使用。

## 快速開始

1. 克隆專案作為 vault
```bash
git clone https://github.com/AgriciDaniel/claude-obsidian
```
2. 進入專案資料夾
```bash
cd claude-obsidian
```
3. 運行設置腳本
```bash
bash bin/setup-vault.sh
```
4. 在 Obsidian 中打開資料夾
```bash
Manage Vaults → Open folder as vault → select claude-obsidian/
```
5. 在 Claude Code 中輸入 /wiki 開始使用
```bash
/wiki
```

## 架構分析

claude-obsidian 採用的是一個以 Claude AI 為核心的架構，設計目的是為了自動化筆記的整理和維護。使用者的資料來源透過指令輸入，Claude 會自動提取資料並生成維基頁面。這種設計使得使用者不需要手動進行資料整理，降低了使用門檻。

資料流方面，Claude 會在每次會話結束時更新熱緩存，確保下次使用時能夠快速獲取上下文。這樣的架構選擇使得使用者能夠在不同會話中保持連貫性，並且能夠隨時獲得最新的資訊。選擇使用 Shell 作為主要語言，因為它能夠快速執行系統指令，並且對於自動化任務非常友好。

這樣的設計雖然簡單，但在處理大量資料時可能會遇到性能瓶頸，尤其是在資料量龐大的情況下。

> ### [Youtube Demo](https://www.youtube.com/watch?v=a2hgayvr-H4)
> 
>   
> 
> You drop sources. Claude reads them, extracts entities and concepts, updates cross-references, and files everything into a structured Obsidian vault. The wiki gets richer with every ingest.
> 
> You ask questions. Claude reads the hot cache (recent context), scans the index, drills into relevant pages, and synthesizes an answer. It cites specific wiki pages, not training data.
> 
> You lint. Claude finds orphans, dead links, stale claims, and missing cross-references. Your wiki stays healthy without manual cleanup.
> 
> At the end of every session, Claude updates a hot cache. The next session starts with full recent context, no recap needed.
