## 專案簡介

claude-obsidian 是一個結合 Claude 和 Obsidian 的知識管理工具，旨在自動化筆記的創建、組織和維護。用戶只需將來源文件放入，Claude 會自動提取實體和概念，並更新交叉引用，形成結構化的 wiki。使用者可以透過指令 `/wiki` 開始設置，或使用 `/save` 將對話保存為 wiki 筆記。這個工具的賣點在於它的自動化能力，能夠在用戶不需手動整理的情況下，持續增長和優化知識庫。技術上，claude-obsidian 基於 Karpathy 的 LLM Wiki 模式，並集成了 DragonScale 記憶層，支持多種模型，讓知識管理更為高效。

這個專案的依賴相對輕量，主要使用 Python 和 Shell，適合快速上手和部署。與其他 Obsidian 插件相比，claude-obsidian 不僅僅是回答問題的聊天界面，而是一個主動的知識引擎，能夠自動組織筆記並進行自主研究。這使得它在需要持續更新和維護知識庫的場景中，特別適合。使用者可能會遇到的問題包括命令未被發現的情況，這在使用 git clone 時較為常見。這個專案目前處於穩定階段，值得在個人或小型團隊中試用，未來可能會持續增強其記憶和自動化功能。

**技術棧**：`Python` · `Shell` · `Makefile`

## 重點功能

- 自動組織筆記 — Claude 能夠自動創建實體和概念，並建立交叉引用，無需手動整理。
- 持久的熱緩存 — 每次會話結束時，Claude 會更新熱緩存，保持上下文的連貫性。
- 自主研究 — 使用 `/autoresearch [topic]` 指令，Claude 能夠進行三輪的網路研究，填補知識空白。
- 多模型支持 — 支持 Claude、Gemini、Codex 等多種模型，提供靈活的知識處理能力。
- 健康檢查功能 — 使用 `lint the wiki` 指令，Claude 能夠檢查孤立的筆記、死鏈接和缺失的交叉引用。

## 快速開始

1. 克隆專案並設置 vault
```bash
git clone https://github.com/AgriciDaniel/claude-obsidian && cd claude-obsidian && bash bin/setup-vault.sh
```
2. 在 Claude Code 中安裝插件
```bash
claude plugin install claude-obsidian@claude-obsidian-marketplace
```
3. 檢查插件是否安裝成功
```bash
claude plugin list
```

> 
>   
> 
> You drop sources. Claude reads them, extracts entities and concepts, updates cross-references, and files everything into a structured Obsidian vault. The wiki gets richer with every ingest.
> 
> You ask questions. Claude reads the hot cache (recent context), scans the index, drills into relevant pages, and synthesizes an answer. It cites specific wiki pages, not training data.
> 
> You lint. Claude finds orphans, dead links, stale claims, and missing cross-references. Your wiki stays healthy without manual cleanup.
> 
> At the end of every session, Claude updates a hot cache. The next session starts with full recent context, no recap needed.
