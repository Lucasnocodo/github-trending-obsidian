## 專案簡介

claude-obsidian 是一個結合 Claude 和 Obsidian 的知識伴侶，旨在自動化筆記的創建、組織和維護。使用者只需將來源資料丟入，Claude 會自動提取實體和概念，並更新交叉引用，最終形成結構化的 Obsidian 知識庫。用戶可以透過指令如 `/wiki` 來進行設置，或使用 `/ingest [file]` 來批量處理多個來源，這樣的設計讓知識的增長如同複利般自動化。這個工具的賣點在於其無需手動整理的能力，並且能夠持續更新熱緩存，讓使用者在每次會話中都能獲得最新的上下文。技術上，claude-obsidian 使用 Python 和 Shell，並依賴於 Obsidian 的核心插件來實現其功能，這樣的選擇使得整體架構輕量且易於維護。

與其他類似工具相比，如 Smart Connections 和 Copilot，claude-obsidian 提供了自動組織筆記和多模型支持的能力，這使得它在知識管理上更具優勢。使用者在實際操作中可能會遇到如命令未被發現的問題，但社群活躍度和開發者的回應速度能夠有效解決這些問題。這個專案目前處於穩定階段，適合需要高效知識管理的團隊或個人使用，未來可能會進一步增強其自動化研究的能力。對於需要在 Obsidian 中建立持久知識庫的使用者來說，claude-obsidian 是一個值得考慮的選擇。

**技術棧**：`Python` · `Shell` · `Makefile`

## 重點功能

- 自動組織筆記 — Claude 自動創建實體和概念，無需手動整理。
- 持久熱緩存 — 每次會話都能獲得最新的上下文，無需重複回顧。
- 批量處理來源 — 使用 `/ingest [file]` 指令，快速創建多個 wiki 頁面。
- 自動化研究 — 使用 `/autoresearch [topic]` 指令進行主題研究，並自動整理結果。
- 多模型支持 — 支援 Claude、Gemini、Codex 等多種模型，提升靈活性。

## 快速開始

1. 克隆專案並設置 vault
```bash
git clone https://github.com/AgriciDaniel/claude-obsidian
cd claude-obsidian
bash bin/setup-vault.sh
```
2. 在 Claude Code 中安裝插件
```bash
claude plugin install claude-obsidian@claude-obsidian-marketplace
```
3. 檢查插件是否安裝成功
```bash
claude plugin list
```

## 程式碼範例

```python
# 前置條件
# 使用者需先安裝 Obsidian 和 Local REST API 插件。
claude mcp add-json obsidian-vault '{
  "type": "stdio",

  "args": ["mcp-obsidian"],
  "env": {
    "OBSIDIAN_API_KEY": "your-key",
    "OBSIDIAN_HOST": "127.0.0.1",
    "OBSIDIAN_PORT": "27124",
    "NODE_TLS_REJECT_UNAUTHORIZED": "0"
  }
}' --scope user
# 預期輸出
# 設置成功後，Claude 將能夠直接讀取和寫入 Obsidian vault 的筆記。
```

> ### [Youtube Demo](https://www.youtube.com/watch?v=a2hgayvr-H4)
> 
>   
> 
> You drop sources. Claude reads them, extracts entities and concepts, updates cross-references, and files everything into a structured Obsidian vault. The wiki gets richer with every ingest.
> 
> You ask questions. Claude reads the hot cache (recent context), scans the index, drills into relevant pages, and synthesizes an answer. It cites specific wiki pages, not training data.
> 
> You lint. Claude finds orphans, dead links, stale claims, and missing cross-references. Your wiki stays healthy without manual cleanup.
> 
> At the end of every session, Claude updates a hot cache. The next session starts with full recent context, no recap needed.
