## 專案簡介

Uncodixify 的核心機制是透過一套規則，指導 GPT 在生成 UI 時避免重複的糟糕設計模式。使用者只需將 `uncodixify.md` 包含在提示或系統指令中，GPT 就會受到這些規則的約束，從而生成更符合常規的介面設計。這個工具的賣點在於它不教 GPT 如何設計，而是明確告訴它哪些設計模式應該避免。技術上，這個工具可以作為一個 agent skill 被集成到支持該格式的 AI 編碼代理中，像是 Codex 和 Claude Code，使用者可以透過 `npx skills add cyxzdev/Uncodixfy` 命令輕鬆安裝並使用。相較於其他 UI 生成工具，Uncodixify 主要專注於消除不良設計，而不是提供設計靈感，這使得它在特定場景下更具針對性。

這樣的設計選擇使得它的依賴性極低，幾乎不需要額外的資源來運行。使用者在實際應用中，可能會發現 GPT 生成的 UI 更加符合用戶的預期，特別是在設計需求較高的專案中。這個工具的成熟度尚在 alpha 階段，適合對 UI 設計有高要求的開發者使用，未來可能會隨著社群的反饋而持續優化。對於需要快速生成 UI 的小型團隊或個人開發者來說，Uncodixify 是一個值得考慮的選擇，但在大型專案中，可能需要更多的設計指導。總體而言，這個工具的使用情境包括快速原型設計、UI 測試和教育用途，而不適合完全依賴其生成的設計作為最終產品。

## 重點功能

> # Uncodixify
> 
>   
> 
> GPT is surprisingly bad at UI design.
> 
> If you ask it to generate interfaces long enough, you start noticing the same bad design-patterns repeating.
> 
> Floating cards.  
> Oversized rounded corners.  
> Gradient-heavy dashboards.    
> Decorative labels everywhere.  
> Glass panels.  
> 
> After a while you can recognize a “GPT UI” immediately.
> 
> This file exists to stop that.
> 
> ---
> 
> ## What this is
> 
> `uncodixify.md` is a rule set that forces GPT to stop relying on its usual UI habits.
> 
> Instead of letting it improvise design decisions, the file blocks the patterns it almost always falls back to and pushes it toward more normal interfaces.
> 
> It doesn't try to teach GPT how to design.
> 
> It mostly just tells it what **not** to do.
> 
> ## Using it
> 
> Include `uncodixify.md` in your prompt or system instructions when asking GPT to generate UI.
> 
> Example:
> 
>   
>   
> 
> Before (Typical GPT UI) | After (Uncodixified)
> 
>   
>   
> 
> Before (Typical GPT UI) | After (Uncodixified)
>

## Star 趨勢

> [!abstract]- Stars 成長追蹤
> ```dataviewjs
> const me = dv.page("Repos/cyxzdev--Uncodixfy");
> if (me?.star_history) {
>   const raw = me.star_history.toString();
>   const points = raw.split(",").map(p => { const [d, s] = p.split(":"); return { date: d, stars: parseInt(s) }; }).filter(p => !isNaN(p.stars));
>   if (points.length >= 2) {
>     const max = Math.max(...points.map(p => p.stars));
>     const lines = points.map(p => {
>       const w = Math.round(p.stars / max * 25);
>       return `${p.date} ${"\u2588".repeat(w)}${"\u2591".repeat(25-w)} ${p.stars.toLocaleString()}`;
>     });
>     const first = points[0].stars;
>     const last = points[points.length-1].stars;
>     const growth = first > 0 ? Math.round((last - first) / first * 100) : 0;
>     lines.push(`\n**成長** +${(last-first).toLocaleString()} stars（${growth}%）in ${points.length} snapshots`);
>     // 趨勢方向偵測
>     if (points.length >= 3) {
>       const mid = Math.floor(points.length / 2);
>       const fh = points.slice(0, mid), sh = points.slice(mid);
>       const rateF = fh.length > 1 ? (fh[fh.length-1].stars - fh[0].stars) / Math.max(1, (new Date(fh[fh.length-1].date) - new Date(fh[0].date)) / 86400000) : 0;
>       const rateS = sh.length > 1 ? (sh[sh.length-1].stars - sh[0].stars) / Math.max(1, (new Date(sh[sh.length-1].date) - new Date(sh[0].date)) / 86400000) : 0;
>       const ratio = rateF > 0 ? rateS / rateF : rateS > 0 ? 2 : 1;
>       const dir = ratio > 1.3 ? "Rising（加速中）" : ratio < 0.7 ? "Cooling（降溫中）" : "Stable（穩定）";
