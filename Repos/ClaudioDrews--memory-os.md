## 專案簡介

Memory OS 是一個針對 Hermes Agent 的七層記憶操作系統，旨在解決 AI 代理在多次會話中無法記住上下文的問題。它通過結構化的事實儲存、智能上下文注入和自動策劃的維基系統來實現持久記憶。用戶可以透過 `curl -sSL https://raw.githubusercontent.com/ClaudioDrews/memory-os/main/setup.sh | bash` 指令，快速安裝整個系統，並且不需要任何雲端訂閱，完全在本地運行。這個系統的核心在於第七層的「真實性層級」，確保注入的記憶被 AI 代理視為權威資料，避免重複查詢和浪費資源。技術上，Memory OS 使用 Qdrant 作為向量資料庫，並結合 Redis 和 ARQ Worker 進行資料管理，這樣的架構設計使得系統能夠高效地處理大量的記憶資料。相較於其他雲端優先的記憶解決方案，Memory OS 提供了更高的隱私性和靈活性，因為所有資料都保存在本地，並且支持多種 LLM 提供者。這使得它在需要私密記憶基礎設施的情境下，成為一個理想的選擇。

**技術棧**：`Python 3.11` · `Docker` · `Redis` · `Qdrant`

## 重點功能

- 七層記憶架構 — 包括工作空間、會話、結構化事實、跨會話回憶等層級，確保記憶的完整性和可用性。
- 智能上下文注入 — 自動將相關上下文注入到每次會話中，減少重複查詢的需求。
- 本地運行 — 完全在用戶的機器上運行，無需雲端訂閱，保障數據隱私。
- 支持多種 LLM 提供者 — 兼容 OpenRouter、OpenAI、Anthropic 等多個 LLM，靈活性高。
- 自動策劃維基 — 系統會自動整理和更新知識庫，方便用戶查閱歷史資料。

## 架構分析

Memory OS 的架構由七層組成，從工作空間到向量資料庫，確保記憶的捕捉、儲存和注入。每一層都有其特定的功能，例如使用 SQLite 和 FTS5 進行會話管理，Qdrant 作為向量資料庫進行高效檢索。這樣的設計使得系統能夠在本地運行，避免了雲端解決方案的隱私問題。選擇本地化架構的代價是需要用戶自行管理基礎設施，但這也帶來了更高的靈活性和控制權。整體架構的擴展性良好，但在高負載情況下，可能需要更強的硬體支持來維持效能。

> ## Why Layer 7 is the most important layer
> 
> Layers 1-6 ensure memory is **captured, stored, and injected**. Layer 7 ensures the injected memory is **used**.
> 
> Without the Ground Truth hierarchy:
> - Qdrant points are injected but the agent calls the Qdrant API to verify them
> - Fabric entries are injected but the agent runs `fabric_recall` to re-find them  
> - Session history is injected but the agent runs `session_search` to re-discover it
> - Facts are injected but the agent probes `fact_store` to confirm them
> 
> The result: **memory-zero behavior** despite perfect injection. Every rediscovery burns tokens, context, and time.
> 
> → **[Read Layer 7: Ground Truth Hierarchy](layers/07-ground-truth.md)** — the critical fix.
>
