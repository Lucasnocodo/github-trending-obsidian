## 專案簡介

Memory OS 是一個針對 Hermes Agent 設計的七層記憶操作系統，旨在解決 AI 記憶不持久的問題。它將用戶的對話、決策和結構化事實進行捕獲、存儲和注入，並透過 Qdrant 提供向量搜索功能。用戶可以透過簡單的指令 `curl -sSL https://raw.githubusercontent.com/ClaudioDrews/memory-os/main/setup.sh | bash` 一鍵安裝整個系統，無需雲端訂閱，並且支持多種 LLM 提供者。這個系統的關鍵在於第七層的「真實性層級」，確保注入的記憶被實際使用，避免了重複查詢和浪費 token 的問題。

技術上，Memory OS 使用了 Redis 和 Qdrant 的結合，並且透過自動化的知識管道進行持續更新，這些設計使得它在本地運行時能夠提供高效能的記憶管理。與其他雲端記憶解決方案相比，Memory OS 提供了更高的隱私性和靈活性，特別適合需要本地化記憶基礎設施的用戶。使用者在實際應用中可能會遇到如 Docker 安裝的問題，但社群的活躍度和持續的更新能夠有效解決這些問題。整體來看，這是一個適合對記憶管理有高需求的開發者，尤其是那些希望在本地環境中運行 AI 的團隊。

**技術棧**：`Python 3.11` · `Docker` · `Redis` · `Qdrant`

## 重點功能

- 七層記憶架構 — 包含工作區、會話、結構化事實、跨會話記憶、向量數據庫、自動化維基和真實性層級。
- 本地記憶基礎設施 — 完全在用戶的機器上運行，無需雲端訂閱。
- 智能上下文注入 — 自動將相關記憶注入到對話中，提升 AI 的上下文理解能力。
- 結構化事實存儲 — 透過信任評分和反饋循環來管理事實，確保資料的準確性。
- Qdrant 向量搜索 — 提供高效的語義搜索功能，支持多層次的回退機制。

## 架構分析

Memory OS 的架構由七個層次組成，第一層為工作區，存儲用戶的基本資料；第二層為會話層，利用 SQLite 和 FTS5 提供全文本搜索；第三層為結構化事實，結合信任評分進行資料管理；第四層為跨會話的 Fabric 層，支持多來源的記憶注入；第五層為向量數據庫 Qdrant，提供高效的語義檢索；第六層為自動化維基，持續更新知識；第七層為真實性層級，確保注入的記憶被正確使用。這樣的設計使得系統能夠在本地運行，避免了雲端依賴的風險，並且每一層都能夠有效地提升記憶的管理和使用效率。選擇這樣的架構是為了提供一個全面的記憶解決方案，但也帶來了系統複雜度的增加，對於新手用戶可能需要更多的學習和適應。

## 技術深入分析

Memory OS 的核心技術機制在於其七層記憶架構，這種設計使得記憶的捕獲、存儲和使用都能夠高效進行。每一層都有其特定的功能，如第一層的工作區用於存儲用戶的基本資料，第二層的會話層則利用 SQLite 和 FTS5 提供全文本搜索，這樣的設計使得用戶能夠快速檢索過去的對話。效能方面，系統能夠處理大量的資料，並且在使用 Qdrant 向量數據庫時，提供了高效的語義檢索功能。選擇 Python 和 Docker 作為主要技術棧，讓開發者能夠快速上手並進行部署，但這也帶來了對 Docker 環境的依賴。設計上的取捨在於本地運行的隱私性和靈活性，與雲端解決方案相比，雖然提供了更高的安全性，但也增加了系統的複雜度。技術風險方面，對 Docker 的依賴可能在某些環境下導致性能瓶頸，特別是在資源有限的情況下。整合方面，與主流的 LLM 提供者兼容性良好，但在實際使用中，可能會遇到安裝和配置的挑戰，這需要用戶具備一定的技術背景。

> # Memory OS — Hermes Agent Memory Operating System
> 
> > **Your agent finally stops forgetting.**  \

> 
> Memory OS turns Hermes Agent into a real long-term collaborator — one that remembers your projects, your decisions, your reasoning, and brings exactly the right context back at exactly the right moment. Like talking to a colleague who was there for every session.
>

>

> ## Why Layer 7 is the most important layer
> 
> Layers 1-6 ensure memory is **captured, stored, and injected**. Layer 7 ensures the injected memory is **used**.
> 
> Without the Ground Truth hierarchy:
> - Qdrant points are injected but the agent calls the Qdrant API to verify them
> - Fabric entries are injected but the agent runs `fabric_recall` to re-find them  
> - Session history is injected but the agent runs `session_search` to re-discover it
> - Facts are injected but the agent probes `fact_store` to confirm them
> 
> The result: **memory-zero behavior** despite perfect injection. Every rediscovery burns tokens, context, and time.
> 
> → **[Read Layer 7: Ground Truth Hierarchy](layers/07-ground-truth.md)** — the critical fix.
>

> 
>

> 
> | Aspect | Stock Hermes | Memory OS |

> | Workspace memory | MEMORY.md + USER.md | + CREATIVE.md + intelligent injection |
