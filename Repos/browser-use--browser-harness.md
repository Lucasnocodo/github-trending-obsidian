## 專案簡介

Browser Harness 是一個輕量級的 CDP (Chrome DevTools Protocol) 工具，讓 LLM 能夠直接與真實瀏覽器互動，實現完全自動化的網頁操作。使用者只需透過一個 WebSocket 連接到 Chrome，LLM 便能在執行過程中自動生成所需的輔助程式碼，進而完成任務。例如，當 LLM 需要上傳檔案時，會自動創建相應的輔助函式，並在執行時進行調用。這種自我修復的特性使得使用者幾乎不需要手動介入，從而提升了效率。核心的指令是 `browser-harness`，用於啟動這個工具。這個工具的設計目的是為了提供最大的自由度，讓 LLM 可以在瀏覽器中執行各種複雜的任務，而不受限於傳統的 API 或框架。

技術上，它依賴於 Python 3.11 及以上版本，並使用了 `cdp-use` 和 `fetch-use` 等庫來實現與瀏覽器的通信。這樣的選擇使得整體架構輕量且易於擴展，適合快速開發和迭代。與其他自動化工具相比，如 Selenium 或 Puppeteer，Browser Harness 的優勢在於其自我學習能力，能夠在每次執行中不斷優化自身的操作流程，這在處理動態網站或複雜表單時特別有用。使用者在實際操作中可能會遇到一些限制，例如對特定框架（如 Ember.js）的支援不足，這在熱門 Issues 中有提到。這個專案目前仍處於活躍開發中，社群的回應速度和貢獻活躍度都相對良好，適合希望在自動化任務中尋求更高自由度的開發者。對於小型團隊或個人專案，這個工具的學習曲線相對平緩，值得一試。

**技術棧**：`Python 3.11` · `cdp-use 1.4.5` · `fetch-use 0.4.0` · `pillow 12.2.0` · `websockets 15.0.1`

## 重點功能

- 自我修復能力 — LLM 能夠在執行過程中自動生成所需的輔助程式碼。
- 輕量級架構 — 僅需一個 WebSocket 連接到 Chrome，無需額外的中介層。
- 支持多種瀏覽器操作 — 能夠處理文件上傳、表單填寫等複雜任務。
- 即時編輯能力 — 使用者可以在執行過程中修改和擴展功能。
- 免費雲端服務 — 提供 3 個並行瀏覽器的免費層，無需信用卡。

## 架構分析

Browser Harness 採用輕量級的單一 WebSocket 連接架構，這樣的設計使得 LLM 能夠直接與 Chrome 瀏覽器進行交互，減少了中介層的延遲。資料流中，LLM 透過 WebSocket 發送指令，Chrome 接收並執行，並將結果回傳給 LLM。這種設計的好處在於能夠快速響應用戶需求，但缺點是對於某些複雜的操作可能需要額外的輔助程式碼。整體架構的擴展性良好，能夠支持多個並行操作，但在高流量使用情境下可能會遇到性能瓶頸。由於依賴於 Chrome 的遠端調試功能，這也可能在某些環境中造成限制。

## 技術深入分析

Browser Harness 的核心技術機制是通過 WebSocket 直接與 Chrome 瀏覽器進行通信，這使得 LLM 能夠即時發送指令並接收回應。這種架構不僅減少了延遲，還允許 LLM 在執行過程中動態生成所需的輔助程式碼，從而提高了操作的靈活性。效能上，這個工具能夠處理多個並行的瀏覽器實例，具體的性能數據尚未提供，但在高流量情境下可能會出現瓶頸。選擇 Python 作為開發語言使得這個工具易於擴展，但也可能面臨 Python 的性能限制。依賴於 Chrome 的遠端調試功能，這使得其在某些環境中可能會受到限制。設計上，這個工具的依賴樹相對簡單，主要依賴於幾個輕量級的庫，這降低了維護成本。與主流框架的整合難度較低，特別是在 CI/CD pipeline 中，能夠輕鬆集成進行自動化測試和部署。

## README 摘錄

> [!info]- 展開查看原文 README
> # Browser Harness ♞
> 
> Connect an LLM directly to your real browser with a thin, editable CDP harness. For browser tasks where you need **complete freedom**.
> 
> One websocket to Chrome, nothing between. The agent writes what's missing during execution. The harness improves itself every run.
> 
> ```
>   ● agent: wants to upload a file
>   │
>   ● agent-workspace/agent_helpers.py → helper missing
>   │
>   ● agent writes it                         agent_helpers.py
>   │                                                       + custom helper
>   ✓ file uploaded
> ```
> 
> **You will never use the browser again.**
> 
> ## Setup prompt
> 
> Paste into Claude Code or Codex:
> 
> ```text
> Set up https://github.com/browser-use/browser-harness for me.
> 
> Read `install.md` and follow the steps to install browser-harness and connect it to my browser.
> ```
> 
> The agent will open `chrome://inspect/#remote-debugging`. Tick the checkbox so the agent can connect to your browser:
> 
> Click Allow when the per-attach popup appears (Chrome 144+):
> 
> See [agent-workspace/domain-skills/](agent-workspace/domain-skills/) for example tasks.
> 
> ## Free Browser Use Cloud browsers
> 
> Stealth, sub-agents, or headless deployment.
> **Browser Use Cloud free tier: 3 concurrent browsers, proxies, captcha solving, and more. No card required.**
> 
> - Grab a key at [cloud.browser-use.com/new-api-key](https://cloud.browser-use.com/new-api-key)
> - Or let the agent sign up itself via [docs.browser-use.com/llms.txt](https://docs.browser-use.com/llms.txt) (setup flow + challenge context included).
> 
> ## Architecture (~1k lines across 4 core files)
> 
> - `install.md` — first-time install and browser bootstrap
> - `SKILL.md` — day-to-day usage
> - `src/browser_harness/` — protected core package
> - `agent-workspace/agent_helpers.py` — helper code the agent edits
> - `agent-workspace/domain-skills/` — reusable site-specific skills the agent edits
> 
> ## Contributing
> 
> PRs and improvements welcome. The best way to help: **contribute a new domain skill** under [agent-workspace/domain-skills/](agent-workspace/domain-skills/) for a site or task you use often (LinkedIn outreach, ordering on Amazon, filing expenses, etc.). Each skill teaches the agent the selectors, flows, and edge cases it would otherwise have to rediscover.
> 
> - **Skills are written by the harness, not by you.** Just run your task with the agent — when it figures something non-obvious out, it files the skill itself (see [SKILL.md](SKILL.md)). Please don't hand-author skill files; agent-generated ones reflect what actually works in the browser.
> - Open a PR with the generated `agent-workspace/domain-skills//` folder — small and focused is great.
> - Bug fixes, docs tweaks, and helper improvements are equally welcome.
> - Browse existing skills (`github/`, `linkedin/`, `amazon/`, ...) to see the shape.
> 
> If you're not sure where to start, open an issue and we'll point you somewhere useful.
>
