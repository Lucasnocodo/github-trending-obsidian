## 專案簡介

PDFx 是一個開放且向後相容的 PDF 擴展，能將多個文件打包成一個單一的 PDF 文件，並提供一個簡單的桌面查看器，支援 macOS、Windows 和 Linux。用戶可以將 PDF 文件、圖片或資料夾拖放到應用中，這些文件會以頁面的橫向條帶形式呈現，並可以垂直堆疊。用戶可以重新排序或移除文件，然後選擇「Export PDF」將整個集合保存為一個文件。這種設計使得 PDFx 能在不改變原有 PDF 文件的情況下，提供多文件的管理功能。核心技術使用 Electron、Vite、TypeScript 和 React，並利用 pdf.js 進行 PDF 渲染，pdf-lib 負責文件的組裝。這樣的架構選擇使得開發過程中能夠快速迭代，並且保持跨平台的兼容性。

相較於傳統的 PDF 工具，PDFx 提供了更靈活的文件管理方式，特別適合需要處理多個文件的用戶。與 Adobe Acrobat 等傳統 PDF 編輯工具相比，PDFx 的操作界面更為簡潔，且無需高額的授權費用。儘管如此，PDFx 目前仍處於 beta 階段，可能會遇到一些穩定性問題，例如 OCR 功能在某些情況下會導致文本損壞。未來的發展方向可能會集中在增強 OCR 功能和優化用戶體驗上。適合需要將多個文件整合為一個 PDF 的用戶，尤其是小型團隊或個人使用者。對於大型企業或需要高穩定性的環境，建議等待進一步的穩定版本。

**技術棧**：`Electron` · `Vite` · `TypeScript` · `React` · `pdf.js` · `pdf-lib`

## 重點功能

- 多文件整合 — 支援將多個 PDF、圖片或資料夾拖放進應用中，並以單一 PDF 文件輸出。
- 簡易操作界面 — 用戶可透過拖放和簡單的按鈕操作來管理文件，無需複雜的設置。
- 跨平台支援 — 提供 macOS、Windows 和 Linux 的桌面應用，方便不同環境的使用者。
- OCR 功能 — 支援光學字符識別，能夠將掃描的文件轉換為可搜索的文本。
- JSON 清單嵌入 — 使用嵌入的 JSON 清單來管理多文件結構，保持 PDF 的有效性。

## 架構分析

PDFx 採用 Electron 作為桌面應用框架，這使得它能夠在多個操作系統上運行。資料流方面，使用 pdf.js 進行 PDF 渲染，並利用 pdf-lib 來組裝多個文件。這種設計讓開發者能夠快速迭代，並保持良好的跨平台兼容性。

選擇 Electron 也意味著能夠使用網頁技術進行開發，但同時也帶來了較大的應用包體積。擴展性方面，由於依賴於 Electron 和 React，未來若需增加功能，開發者可以利用現有的生態系統進行擴展，但可能會面臨性能瓶頸。整體而言，這種架構適合小型到中型的應用，但在處理大型文件時可能會遇到性能問題。

## 技術深入分析

PDFx 的核心技術機制在於其使用 Electron 作為應用框架，這使得它能夠利用網頁技術進行開發，並在多個平台上運行。資料流方面，PDFx 使用 pdf.js 進行 PDF 的渲染，這是一個成熟的開源庫，能夠處理各種 PDF 文件。pdf-lib 則負責將多個文件組合成一個 PDF，這種設計選擇使得開發者能夠快速實現功能，並保持良好的性能。關於效能，PDFx 在處理小型文件時表現良好，但在處理大型 PDF 文件時可能會遇到性能瓶頸，特別是在 OCR 功能啟用的情況下。設計上選擇 Electron 也意味著應用的體積會較大，這可能影響到用戶的下載和安裝體驗。技術風險方面，隨著用戶數量的增加，可能會出現穩定性問題，特別是在多文件處理時。整合方面，PDFx 可與現有的開發工具鏈（如 VS Code）良好整合，並能夠在 CI/CD pipeline 中使用，但需要注意其依賴的 Electron 版本更新可能會影響穩定性。

> ## What it is
> 
> PDFx is an open, backwards compatible extension of PDF that bundles many documents into a single file, plus a minimal desktop viewer for macOS, Windows, and Linux.
> 
> The files produced are fully valid PDFs: open them anywhere and every page shows in sequence. Open them in PDFx and it splits back into the original documents. Plain single PDFs work as they are.
> 
> Drag and drop `.pdf` files, images or folders anywhere in the window. Each document renders as a horizontal strip of pages, and documents stack vertically. Reorder or remove them, then **Export PDF** to save the whole collection as one file.
> 
> See [SPEC.md](SPEC.md) for the format. It is short: the entire trick is one embedded JSON manifest.
> 
> ## How to run
> 
> Built with Electron, Vite, TypeScript, and React. PDF rendering by [pdf.js](https://mozilla.github.io/pdf.js/), assembly by [pdf-lib](https://pdf-lib.js.org/).
> 
> ```bash
> yarn              # install
> yarn dev          # run in development
> yarn dev:web      # run on the web
> yarn build:mac    # build:win for windows, build:linux for linux
> ```
> 
> *Linux packaging notes (formats, Flatpak, prerequisites) live in [docs/LINUX_SETUP.md](docs/LINUX_SETUP.md).*
