## 專案簡介

PDFx 是一個向後兼容的 PDF 擴展，允許將多個文件打包成單一文件，並提供一個簡易的桌面查看器。使用者可以將 `.pdf` 或 `.pdfx` 文件拖放到應用程式中，系統會將其分解為原始文件，並以水平頁面條帶的形式顯示。這樣的設計使得使用者能夠輕鬆地重新排列或移除文件，然後通過「Export PDF」功能將整個集合保存為一個文件。其核心機制是透過嵌入的 JSON 清單來實現文件的組合，這樣的設計不僅保持了 PDF 的完整性，還提供了靈活的文件管理功能。主要依賴於 Electron、Vite、TypeScript 和 React，PDF 的渲染則使用了 pdf.js，組合則由 pdf-lib 處理。

這樣的技術選型使得 PDFx 在跨平台上具備良好的性能，並且能夠支持多種操作系統。與傳統的 PDF 工具相比，PDFx 提供了更高的靈活性，因為它不僅支持單一文件的查看，還能夠將多個文件整合為一個，這在需要處理大量文檔的情況下尤為重要。雖然目前仍有一些開放的問題，例如 OCR 可能會導致文本損壞，但整體的功能設計已經相當成熟。這個專案適合需要在文件管理上有更高需求的使用者，特別是在需要將多個文檔整合為一個文件的場景下。未來可能會進一步優化 OCR 和搜索功能，以提升使用者體驗。

**技術棧**：`Electron` · `Vite` · `TypeScript` · `React` · `pdf.js` · `pdf-lib`

## 重點功能

- 多文件打包 — 允許將多個 PDF 文件打包為一個 `.pdfx` 文件，並透過 JSON 清單管理。
- 跨平台支持 — 提供 macOS、Windows 和 Linux 的桌面查看器，方便不同系統的使用者使用。
- 即時預覽 — 拖放文件後即時顯示頁面，方便用戶快速檢視和管理文件。
- 簡易導出功能 — 透過「Export PDF」按鈕將整個集合導出為單一文件，簡化操作流程。
- OCR 支持 — 提供 OCR 功能，能夠識別圖像中的文本，提升搜索能力。

## 架構分析

PDFx 的架構基於 Electron 平台，這使得它能夠在多個操作系統上運行。核心功能是將多個 PDF 文件打包成一個文件，這是通過嵌入 JSON 清單來實現的。這樣的設計使得 PDFx 在保持 PDF 文件完整性的同時，能夠提供靈活的文件管理功能。

資料流方面，使用 pdf.js 進行 PDF 的渲染，而 pdf-lib 負責文件的組合。選擇 Electron 作為架構的代價是需要額外的資源來支持跨平台的運行，但這也帶來了更廣泛的使用者基礎。擴展性方面，未來可能會增加更多的文件格式支持，但目前的設計主要針對 PDF 文件。

## 技術深入分析

PDFx 的核心技術機制是將多個 PDF 文件打包為一個文件，這是通過嵌入 JSON 清單來實現的。這樣的設計不僅保持了 PDF 文件的完整性，還提供了靈活的文件管理功能。效能方面，PDFx 能夠在多平台上運行，但對於大型 PDF 文件的處理速度仍有提升空間。選擇 Electron 作為架構的原因是其跨平台的能力，但這也意味著需要額外的資源來支持。技術風險方面，OCR 功能的穩定性尚未得到充分驗證，可能會在未來的版本中引發問題。整合方面，PDFx 與現有的 PDF 工具鏈相容性良好，但在處理大型文件時可能會遇到性能瓶頸。

> ## What it is
> 
> PDFx is an open, backwards compatible extension of PDF that bundles many documents into a single file, plus a minimal desktop viewer for macOS, Windows, and Linux.
> 
> A `.pdfx` file is a fully valid PDF: open it anywhere and every page shows in sequence. Open it in PDFx and it splits back into the original documents. Plain single PDFs work as they are.
> 
> Drag and drop `.pdf` or `.pdfx` files anywhere in the window. Each document renders as a horizontal strip of pages, and documents stack vertically. Reorder or remove them, then **Export PDF** to save the whole collection as one file.
> 
> See [SPEC.md](SPEC.md) for the format. It is short: the entire trick is one embedded JSON manifest.
> 
> ## How to run
> 
> Built with Electron, Vite, TypeScript, and React. PDF rendering by [pdf.js](https://mozilla.github.io/pdf.js/), assembly by [pdf-lib](https://pdf-lib.js.org/).
> 
> ```bash
> yarn              # install
> yarn dev          # run in development
> yarn build:mac    # package for macOS
> yarn build:win    # package for Windows
> yarn build:linux  # package for Linux
> ```
> 
> *Linux packaging notes (formats, Flatpak, prerequisites) live in [docs/LINUX_SETUP.md](docs/LINUX_SETUP.md).*
