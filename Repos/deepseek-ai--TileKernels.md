## 專案簡介

TileKernels 是一個基於 TileLang 的 GPU 核心庫，專注於優化大型語言模型（LLM）的運算性能。使用者可以透過簡單的 Python 語法來調用這些高效能的核心，實現如 Top-k 專家選擇、量化、轉置等操作。這些核心已在內部訓練和推論中得到應用，並且設計上考慮了計算強度和記憶體帶寬的極限。核心功能包括 Gating、MoE Routing 和 Engram，這些功能能夠在訓練過程中提供更高的效率和靈活性。使用者只需安裝 Python 3.10 以上和 PyTorch 2.10 以上的版本，並確保擁有 NVIDIA SM90 或 SM100 架構的 GPU 來運行這些核心。

這個專案的賣點在於它能夠讓開發者快速構建和優化 LLM 的運算，並且支持自動優化和敏捷開發。相較於其他 GPU 核心庫，如 NVIDIA 的 cuDNN，TileKernels 提供了更高層次的抽象，讓開發者能夠更容易地進行模型的調整和優化。這樣的設計使得開發者能夠專注於模型本身，而不必深入底層的 GPU 編程。雖然目前專案仍處於 Alpha 階段，但其潛力和功能性已經吸引了不少開發者的注意。對於需要高效能 LLM 訓練的團隊，這是一個值得考慮的工具，但在使用時需注意其文檔和測試的完善程度。

**技術棧**：`TileLang` · `PyTorch 2.10` · `CUDA Toolkit 13.1`

## 重點功能

- Gating — 提供 Top-k 專家選擇和評分功能，支持 Mixture of Experts 路由。
- MoE Routing — 支援 Token-to-expert 映射，融合擴展/縮減和權重正規化。
- Quantization — 支持 FP8/FP4/E5M6 的量化操作，能夠進行每個 token、每個區塊和每個通道的量化。
- Transpose — 提供批量轉置操作，優化記憶體使用。
- Engram — 包含融合 RMSNorm 的 Engram gating 核心，支持前向/反向傳播和權重梯度縮減。
- Manifold HyperConnection — 提供超連接核心，包括 Sinkhorn 正規化和混合拆分/應用。
- Modeling — 提供高層次的 `torch.autograd.Function` 包裝，將低層次核心組合成可訓練的層。

## 架構分析

TileKernels 採用模組化設計，將不同的 GPU 核心功能分為多個子模組，如 MoE、量化和轉置等。這樣的設計使得開發者可以根據需求選擇性地使用特定的核心，降低了整體的複雜度。每個模組都針對特定的運算需求進行優化，並且可以輕鬆整合到 PyTorch 的訓練流程中。這種架構的代價是需要開發者對每個模組的使用場景有一定的了解，否則可能無法充分發揮性能。擴展性方面，隨著新功能的加入，可能會增加維護的負擔，但目前的模組化設計有助於減少這種風險。

## 技術深入分析

TileKernels 的核心技術在於使用 TileLang 這種專門為 GPU 核心設計的語言，這使得開發者能夠以 Python 的語法來表達高效能的運算。這樣的設計不僅提高了開發效率，還能自動優化運算性能。專案的模組化結構允許開發者根據需求選擇不同的核心功能，如 Gating 和 MoE Routing，這些功能能夠在訓練過程中提供更高的效率。效能方面，專案針對 NVIDIA SM90 和 SM100 架構進行了優化，能夠充分利用這些硬體的計算能力。

設計上，選擇 TileLang 而非傳統的 CUDA 編程，降低了開發的複雜度，並提高了可讀性。這種選擇的代價是可能無法完全控制底層的硬體性能，但對於大多數開發者來說，這樣的折衷是值得的。在技術風險方面，專案仍處於 Alpha 階段，未來可能會面臨 API 穩定性和功能擴展的挑戰。整合方面，TileKernels 與 PyTorch 的整合相對簡單，開發者可以輕鬆地將其加入到現有的訓練流程中，並且支持常見的 CI/CD 工具，降低了導入的摩擦。

## README 摘錄

> [!info]- 展開查看原文 README
> # Tile Kernels
> 
> Optimized GPU kernels for LLM operations, built with [TileLang](https://github.com/tile-ai/tilelang). TileLang is a domain-specific language for expressing high-performance GPU kernels in Python, featuring easy migration, agile development, and automatic optimization.
> 
> Most kernels in this project approach the limit of hardware performance regarding the compute intensity and memory bandwidth. Some of them have already been used in internal training and inference scenarios. However, they do not represent best practices and we are actively working on improving the code quality and documentation.
> 
> ## Features
> 
> - **Gating** — Top-k expert selection and scoring for Mixture of Experts routing
> - **MoE Routing** — Token-to-expert mapping, fused expansion/reduction and weight normalization
> - **Quantization** — Per-token, per-block, and per-channel FP8/FP4/E5M6 casting with fused SwiGLU+quantization ops
> - **Transpose** — Batched transpose operations
> - **Engram** — Engram gating kernels with fused RMSNorm, forward/backward passes and weight gradient reduction
> - **Manifold HyperConnection** — Hyper-connection kernels including Sinkhorn normalization and mix splitting/application
> - **Modeling** — High-level `torch.autograd.Function` wrappers composing low-level kernels into trainable layers (engram gate, mHC pipeline)
> 
> ## Requirements
> 
> - Python 3.10 or higher
> - PyTorch 2.10 or higher
> - TileLang 0.1.9 or higher
> - NVIDIA SM90 or SM100 architecture GPU
> - CUDA Toolkit 13.1 or higher
>
