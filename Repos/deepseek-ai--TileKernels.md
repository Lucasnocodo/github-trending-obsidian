## 專案簡介

TileKernels 是一個基於 TileLang 的 GPU 核心庫，專注於優化 LLM 操作的效能。使用者可以透過簡單的 Python 語法來定義高效能的 GPU 核心，並利用 TileLang 的特性進行自動優化。這個庫的核心機制包括多種高效能運算，如 Gating、MoE Routing 和 Quantization，能夠處理複雜的模型運算，並且在內部訓練和推論中已經得到應用。使用者只需安裝相應的依賴，便可輕鬆使用這些功能，像是透過 `pip install tile-kernels` 來安裝。這個工具的賣點在於其高效能和易於使用的 API，能夠幫助開發者快速構建和優化 LLM 模型。

技術上，TileKernels 依賴於 PyTorch 和 TileLang，並要求 NVIDIA SM90 或 SM100 架構的 GPU，這使得其在效能上達到硬體的極限。與其他工具相比，如 Hugging Face Transformers，TileKernels 提供了更細緻的控制和優化選項，但可能需要更高的學習成本。實際使用中，開發者可能會遇到文檔不足的問題，因為目前仍在持續改進中。這個專案目前處於 Alpha 階段，適合對 GPU 核心開發有興趣的開發者，尤其是那些希望在 LLM 領域進行深入探索的團隊。未來六個月內，隨著文檔和功能的增強，這個專案有潛力成為 LLM 開發的主流工具之一。

**技術棧**：`TileLang` · `PyTorch 2.10` · `CUDA Toolkit 13.1`

## 重點功能

- Gating — 提供 Top-k 專家選擇和評分功能，適合 Mixture of Experts 路由。
- MoE Routing — 支援令牌到專家的映射，融合擴展/縮減和權重正規化。
- Quantization — 支援每個令牌、每個區塊和每個通道的 FP8/FP4/E5M6 轉換。
- Transpose — 支援批次轉置操作，提升運算效率。
- Engram — 提供融合 RMSNorm 的 Engram gating 核心，支援前向/反向傳播。
- Manifold HyperConnection — 提供超連接核心，包括 Sinkhorn 正規化和混合分割。
- Modeling — 高層次的 `torch.autograd.Function` 包裝器，將低層次核心組合成可訓練的層。

## 架構分析

TileKernels 採用模組化設計，將不同的運算功能分為多個子模組，如 MoE、量化和轉置等。這樣的設計使得開發者可以根據需求選擇性地使用特定的核心，而不必載入整個庫。資料流方面，核心使用 TileLang 來定義 GPU 運算，並透過 PyTorch 的 autograd 系統進行整合。選擇 TileLang 的原因在於其專為高效能運算設計，能夠自動優化運算過程，代價是需要學習新的語言和框架。這樣的架構使得擴展性良好，但對於不熟悉 TileLang 的開發者來說，可能會增加學習成本。

## 技術深入分析

TileKernels 的核心技術機制是基於 TileLang，這是一種專為高效能 GPU 核心設計的領域特定語言。這使得開發者能夠用簡潔的 Python 語法來定義複雜的運算，並自動優化性能。效能方面，這些核心能夠充分利用 NVIDIA GPU 的計算能力，並在內部訓練和推論中已經顯示出良好的效果。設計選擇上，TileKernels 將不同的運算功能模組化，這樣的選擇使得擴展性更好，但也增加了學習成本。技術風險方面，由於目前仍在 Alpha 階段，未來可能會面臨 API 穩定性問題。整合方面，TileKernels 與 PyTorch 的整合相對簡單，但對於不熟悉 TileLang 的開發者來說，可能需要額外的學習時間。

## README 摘錄

> # Tile Kernels
> 
> Optimized GPU kernels for LLM operations, built with [TileLang](https://github.com/tile-ai/tilelang). TileLang is a domain-specific language for expressing high-performance GPU kernels in Python, featuring easy migration, agile development, and automatic optimization.
> 
> Most kernels in this project approach the limit of hardware performance regarding the compute intensity and memory bandwidth. Some of them have already been used in internal training and inference scenarios. However, they do not represent best practices and we are actively working on improving the code quality and documentation.
> 
> ## Features
> 
> - **Gating** — Top-k expert selection and scoring for Mixture of Experts routing
> - **MoE Routing** — Token-to-expert mapping, fused expansion/reduction and weight normalization
> - **Quantization** — Per-token, per-block, and per-channel FP8/FP4/E5M6 casting with fused SwiGLU+quantization ops
> - **Transpose** — Batched transpose operations
> - **Engram** — Engram gating kernels with fused RMSNorm, forward/backward passes and weight gradient reduction
> - **Manifold HyperConnection** — Hyper-connection kernels including Sinkhorn normalization and mix splitting/application
> - **Modeling** — High-level `torch.autograd.Function` wrappers composing low-level kernels into trainable layers (engram gate, mHC pipeline)
> 
> ## Requirements
> 
> - Python 3.10 or higher
> - PyTorch 2.10 or higher
> - TileLang 0.1.9 or higher
> - NVIDIA SM90 or SM100 architecture GPU
> - CUDA Toolkit 13.1 or higher
>
