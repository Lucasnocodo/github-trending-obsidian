## 專案簡介

Duel Agents 是一個整合多種 LLM 模型的工具，透過一個統一的 API 層，讓開發者可以輕鬆地使用不同的模型並選擇最具成本效益的答案。使用者需要先從官方網站獲取 API 金鑰，然後透過 CLI 安裝所需的工具，像是 Claude Code、Cursor 和 OpenClaw。這些工具會自動配置環境變數，並將 LLM 請求路由至 `https://duelagents.com/v1`。這樣的設計不僅簡化了多模型的管理，還能確保使用者不必處理各個模型的 API 差異。

Duel Agents 的核心在於其 SDK，使用 TypeScript 實作，並提供 OpenAI 兼容的接口，讓開發者能夠快速集成到現有的應用中，並且支持 LangChain 和 LlamaIndex 等流行框架。相較於其他工具如 0x0funky/agent-sprite-forge 和 0xGF/boneyard，Duel Agents 提供了更為簡單的安裝流程和更好的多模型支持，特別是在需要快速切換模型的場景下。這個工具的設計考量了開發者的使用習慣，並提供了詳細的錯誤排除指南，讓使用者能夠快速上手。儘管目前仍在 beta 階段，且社群活躍度尚可，但對於需要多模型支持的開發者來說，這是一個值得關注的選擇。

**技術棧**：`TypeScript` · `Node.js 20+` · `Python`

## 重點功能

- 多模型路由 — 透過統一的 API 層，將請求路由至不同的 LLM 模型。
- CLI 安裝工具 — 使用 `npx @duel-agents/install` 命令快速安裝所需工具。
- 環境變數自動配置 — 自動設置 API 金鑰和模型 URL，減少手動配置的麻煩。
- TypeScript SDK — 提供簡單的 TypeScript 客戶端接口，方便開發者集成。
- LangChain 和 LlamaIndex 支援 — 兼容主流 Python 框架，擴展應用場景。

## 快速開始

2. 安裝所有工具

## 架構分析

Duel Agents 採用 monorepo 架構，將不同的功能模組（如 CLI、SDK 和核心邏輯）分開管理。這樣的設計讓每個模組可以獨立開發和測試，降低了整體的維護成本。資料流方面，所有的 LLM 請求都經由 `https://duelagents.com/v1` 進行路由，並使用 API 金鑰進行身份驗證。

這樣的設計確保了安全性和可擴展性，但也意味著使用者必須依賴外部服務的穩定性。選擇 Node.js 作為主要執行環境，因為其非同步特性能夠有效處理大量的 LLM 請求，這在高流量場景下尤為重要。整體而言，這個架構在擴展性和維護性上做了良好的平衡，但在依賴外部 API 的情況下，可能會面臨延遲和可用性的挑戰。

## 已知陷阱

> [!bug] 踩坑才知道的問題

- **[HIGH]** API 金鑰格式不正確時，無法正常使用工具
  - 解法：確保金鑰格式為 `duel_` + 8 字元 + `_` + 32 字元。
- [MEDIUM] 安裝後 OpenClaw 無法啟動
  - 解法：運行 `openclaw config validate` 以驗證配置。
- [MEDIUM] Cursor 仍使用 OpenAI 模型
  - 解法：確認模型覆蓋 URL 和 API 金鑰是否正確。

> ## Requirements

> ## Troubleshooting
> 
> | Symptom | Fix |
> |---------|-----|
> | `Invalid API key format` | Key must be `duel_` + 8 chars + `_` + 32 chars. Create one at the dashboard. |
> | `401` from doctor | Key revoked or subscription inactive. Create a new key on billing/settings. |
> | `Could not reach Duel API` | The proxy at `duelagents.com/v1` must be running. Key format can still be valid; retry later. |
> | OpenClaw won't start | Run `openclaw config validate` after install; restore from `openclaw.json.bak` if needed. |
> | Cursor still uses OpenAI | Confirm model override URL and that the API key field is your `duel_*` key. |
> | Skill copy failed after npm install | Re-run `npm run build` in the repo, or reinstall `@duel-agents/install`. Skills ship inside the package. |
