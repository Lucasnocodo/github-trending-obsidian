## 專案簡介

ShadowBroker 是一個即時的多領域開源情報 (OSINT) 儀表板，將 60 多個即時情報來源整合到一個黑色操作地圖介面中。用戶可以追蹤飛機、船隻、衛星、衝突區域、CCTV 網絡等，所有數據實時更新在同一螢幕上。這樣的設計使得分析師、研究人員和無線電操作員能夠在一個平台上獲取所有公開信號的視圖，並且完全不收集或傳輸用戶數據。核心指令 `docker compose up -d` 可以快速啟動儀表板，並在本地端運行。這個工具的賣點在於其模組化架構，能夠獨立擴展情報後端和前端界面，並且具備安全的運行環境。

使用了 Next.js、MapLibre GL 和 FastAPI 等技術，這些選擇讓它在性能和擴展性上都表現良好。與其他 OSINT 工具相比，ShadowBroker 提供了更為豐富的數據層和即時更新，這使得用戶能夠在快速變化的環境中做出更準確的決策。這個工具適合需要即時情報的專業人士，尤其是在地緣政治和安全分析方面。使用者可能會遇到 Docker 環境配置的問題，但整體上安裝和使用過程相對簡單。隨著社群的活躍度提升，未來可能會有更多功能和數據來源的擴展。

**技術棧**：`Next.js` · `MapLibre GL` · `FastAPI` · `Python`

## 重點功能

- 即時數據整合 — 整合 60+ 個即時情報來源，提供飛機、船隻、衛星等數據的即時更新。
- 模組化架構 — 獨立擴展情報後端和前端界面，提升系統靈活性。
- 安全運行環境 — 以限制的 UID 運行容器，增強安全性。
- 多層數據顯示 — 提供 35+ 可切換的數據層，讓用戶自定義顯示內容。
- 無用戶數據收集 — 儀表板完全在用戶的瀏覽器中運行，無需傳輸用戶數據。

## 架構分析

ShadowBroker 的架構選擇了模組化設計，將前端和後端分開，使其能夠獨立擴展。前端使用 Next.js 和 MapLibre GL，提供即時的數據可視化，而後端則使用 FastAPI 和 Python 處理數據流。這樣的設計使得系統能夠靈活應對不同的數據需求，但也可能導致在高流量時前後端之間的通信延遲。整體架構的輕量化使得用戶能夠快速啟動和運行，但在處理大量數據時，可能會面臨性能瓶頸。擴展性方面，若未來數據來源增加，可能需要重新評估後端的處理能力和數據存儲方案。

## 技術深入分析

ShadowBroker 的核心技術機制在於其即時數據整合能力，使用了多種開源技術來實現數據的快速處理和可視化。前端使用 Next.js 和 MapLibre GL，這使得用戶能夠在瀏覽器中輕鬆地查看和操作數據。後端則依賴 FastAPI 和 Python，這樣的選擇使得數據處理效率高，並且能夠輕鬆擴展。效能方面，儀表板能夠處理大量的即時數據，但在高流量情況下，可能會出現延遲。設計取捨上，選擇了容器化部署，這雖然提高了安全性，但也增加了用戶的學習成本。技術風險方面，若未來數據來源持續增加，可能會影響系統的穩定性。整合方面，與主流 CI/CD 工具的相容性良好，能夠輕鬆融入現有的開發流程。
