## 專案簡介

Mercury-Transforming-Drone 是一款設計精巧的無人機，具備變形能力，能在不同環境中執行任務。它的核心機制是透過簡單的變形機構，將無人機從飛行模式轉換為地面行駛模式，這樣的設計使其能夠在狹窄或不平坦的地形中靈活運動。使用者只需將自動化軟體上傳至 Raspberry Pi 5，並透過命令 `python3 -m venv venv` 和 `pip install -r requirements.txt` 安裝依賴，便可啟動無人機的控制系統。這個工具的賣點在於其多功能性，結合 RGB、深度和熱成像相機，讓使用者能夠在各種環境下進行拍攝和監控。  

技術上，這款無人機使用 Ardupilot 作為飛行控制系統，並透過 GPS 定位進行導航。它的設計考量了輕量化，內部載重艙可承載 1 公斤的物品，這使得它在運輸和拍攝任務中具備高度的靈活性。與其他無人機相比，如 DJI 的 Phantom 系列，Mercury 提供了更高的自定義性和變形能力，適合需要在多變環境中操作的專業使用者。

  

在實際使用中，無人機的效能表現取決於使用者的網路環境，若需遠程控制，建議使用 Tailscale 進行網路設置。這款無人機的社群活動相對活躍，並且有著良好的維護紀錄，這對於未來的功能擴展和問題修正是個正面信號。  

目前專案處於穩定階段，適合中小型團隊進行實驗性質的專案開發。未來六個月內，預期會有更多功能更新，特別是在自動化和遠程控制方面。  

建議對無人機技術有興趣的開發者使用，尤其是那些希望在特定環境中進行拍攝或監控的專業人士。對於初學者，可能需要一些時間來熟悉其設置和操作流程。

**技術棧**：`Python 3` · `Raspberry Pi 5` · `Ardupilot`

## 重點功能

- 內部載重艙 — 可承載 1 公斤的物品，適合運輸小型設備或拍攝器材。
- 變形機構 — 使無人機能在空中與地面模式之間切換，增強靈活性。
- 多種相機配置 — 包括 RGB、深度和熱成像相機，滿足不同拍攝需求。
- 基於 Ardupilot 的自動化控制 — 提供穩定的飛行控制和導航功能。
- 支援遠程控制 — 可透過 Tailscale 設定，實現長距離操作。

這款無人機的架構設計以 Raspberry Pi 5 為核心控制單元，搭配 Ardupilot 作為飛行控制系統。資料流從相機收集影像，經由 Raspberry Pi 處理後發送至使用者端。選擇 Raspberry Pi 作為控制單元的原因在於其強大的擴展性和社群支持，但這也意味著需要額外的配置和維護。無人機的變形機構設計使其能夠在不同地形上運行，這在技術上增加了設計的複雜度。未來擴展性方面，若要增加更多感測器或功能，可能會面臨資源限制。

> ## Software Setup
> 
> To use the software as it is, upload the Autonomy Software folder to a Raspberry Pi 5, using your preferred SCP method. For beginners we recommend [WinSCP](https://winscp.net/eng/download.php).
> 
> Inside the folder in the raspberry pi create a virtual environment and install the dependencies.
> 
> ```bash
> python3 -m venv venv
> source venv/bin/activate
> pip install -r requirements.txt
> ```
> 
> You must run both the Mavproxy Bridge to interface with the flight controller as well as the main software powering the rest of the robot. For that run in two separate temrinals the scripts:
> 
> ```bash
> start_mavproxy.sh
> ```
> ```bash
> run.sh
> ```
> In the terminal you should see the IP addres to be able to control the robot, if you're connected to the same network just copy paste that on your browser.
> 
> If you would like to be able to control it from different networks and at long distances we recommend you setup [Tailscale](https://tailscale.com/) on your devices.
> 
> For more convenience you can setup these scripts to run automatically on startup, and then use other scripts like `restarter.sh` or `killer.sh` to manage them.
