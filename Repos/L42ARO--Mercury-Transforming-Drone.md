## 專案簡介

Mercury-Transforming-Drone 是一個設計用於多功能任務的無人機，具備 1 公斤的內部載貨艙和簡易的變形機制，能夠適應不同的任務需求。用戶需將自主軟體上傳至 Raspberry Pi 5，並透過 Mavproxy 進行飛行控制，這樣的設計使得無人機能夠在各種環境中靈活運作。使用者只需在兩個終端中運行 `start_mavproxy.sh` 和 `run.sh` 指令，即可開始控制無人機，並透過網路連接進行遠端操控。這種設計的優勢在於簡化了無人機的操作流程，讓使用者能夠快速上手。技術上，該專案使用了 Python 來實現自主飛行功能，並依賴於 Ardupilot 和 GPS 進行定位與導航，這樣的選擇使得無人機在精度和穩定性上有良好的表現。

相較於其他無人機專案，如 AlpinDale/parsync 和 BigBodyCobain/Shadowbroker，Mercury 的變形機制和內部載貨設計提供了更大的靈活性，適合需要多樣化任務的用戶。使用者在實際操作中可能會遇到網路連接不穩定的問題，建議使用 Tailscale 來解決遠端控制的需求。該專案目前處於活躍開發階段，未來可能會增加更多功能以支持更複雜的任務。對於小型團隊或個人開發者來說，這是一個值得嘗試的無人機平台，尤其是在需要快速原型開發的情境下。總體而言，Mercury 提供了一個靈活且易於使用的無人機解決方案，適合需要快速部署和多功能操作的場景。

**技術棧**：`Python 3`

## 重點功能

- 內部載貨艙 — 可容納 1 公斤的貨物，適合多種任務。
- 簡易變形機制 — 使無人機能夠適應不同的環境和需求。
- 多種攝影功能 — 配備 RGB、深度和熱成像攝影機，滿足各種監測需求。
- 基於 Ardupilot 的自主飛行 — 提供穩定的飛行控制和導航能力。
- 支援遠端控制 — 可透過網路進行長距離操控，適合多種應用場景。

## 快速開始

1. 安裝 Python 環境
```bash
python3 -m venv venv
```
2. 啟動虛擬環境
```bash
source venv/bin/activate
```
3. 安裝依賴
```bash
pip install -r requirements.txt
```
4. 啟動 Mavproxy
```bash
./start_mavproxy.sh
```
5. 啟動主程式
```bash
./run.sh
```

> ## Software Setup
> 
> To use the software as it is, upload the Autonomy Software folder to a Raspberry Pi 5, using your preferred SCP method. For beginners we recommend [WinSCP](https://winscp.net/eng/download.php).
> 
> Inside the folder in the raspberry pi create a virtual environment and install the dependencies.
> 
> ```bash
> python3 -m venv venv
> source venv/bin/activate
> pip install -r requirements.txt
> ```
> 
> You must run both the Mavproxy Bridge to interface with the flight controller as well as the main software powering the rest of the robot. For that run in two separate temrinals the scripts:
> 
> ```bash
> start_mavproxy.sh
> ```
> ```bash
> run.sh
> ```
> In the terminal you should see the IP addres to be able to control the robot, if you're connected to the same network just copy paste that on your browser.
> 
> If you would like to be able to control it from different networks and at long distances we recommend you setup [Tailscale](https://tailscale.com/) on your devices.
> 
> For more convenience you can setup these scripts to run automatically on startup, and then use other scripts like `restarter.sh` or `killer.sh` to manage them.
