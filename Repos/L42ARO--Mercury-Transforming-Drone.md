## 專案簡介

Mercury 是一款可變形的無人機，設計上強調多功能性和適應性，適合各種應用場景。用戶可以將其組裝並搭載多種感測器，包括 RGB、深度和熱成像攝影機，並利用 Ardupilot 和 GPS 進行導航。使用者只需將自動化軟體上傳至 Raspberry Pi 5，並透過簡單的指令啟動 Mavproxy 和主控制程式，即可開始操控無人機。這個設計的賣點在於其靈活的貨艙設計，能夠容納最多 1 公斤的有效載荷，並且具備簡單的變形機制，讓無人機在不同環境下運作。技術上，Mercury 使用 Python 和 Shell 腳本來管理自動化過程，並依賴於輕量級的 Raspberry Pi 作為控制單元，這樣的選擇使得整體系統的部署更加靈活且易於維護。

與其他無人機相比，例如 DJI 的商業無人機，Mercury 提供了更高的可定制性和開源的靈活性，適合開發者和愛好者進行二次開發。這使得它在小型專案或教育用途上特別有吸引力。使用者需要注意的是，雖然無人機的組裝和操作相對簡單，但仍需要一定的技術背景來進行調試和維護。這個專案目前處於活躍開發階段，社群支持良好，並且沒有開放的問題，顯示出其穩定性。

**技術棧**：`Python` · `Shell` · `Raspberry Pi`

## 重點功能

- 內部貨艙 — 可容納最多 1 公斤的有效載荷，適合多種應用。
- 簡單變形機制 — 使無人機能夠在不同環境中靈活運作。
- 多種感測器 — 包括 RGB、深度和熱成像攝影機，提供多樣的數據收集能力。
- 基於 Raspberry Pi 的控制系統 — 使得部署和維護變得更加靈活。
- 自動化軟體 — 透過簡單的指令啟動 Mavproxy 和主控制程式，方便操作。

## 快速開始

1. 安裝虛擬環境
```bash
python3 -m venv venv
```
2. 啟動虛擬環境
```bash
source venv/bin/activate
```

3. 安裝依賴
```bash
pip install -r requirements.txt
```

```bash
./start_mavproxy.sh
```
5. 啟動主程式
```bash
./run.sh
```

Mercury 的架構設計以 Raspberry Pi 作為控制中心，並使用 Python 和 Shell 腳本來管理自動化過程。這樣的選擇使得系統能夠保持輕量且易於維護。資料流方面，無人機的感測器數據會通過 Mavproxy 進行處理，並最終傳送至用戶的控制介面。選擇 Raspberry Pi 而非更強大的控制器，雖然降低了計算能力，但大幅提升了成本效益和可接近性。擴展性方面，使用者可以根據需求添加額外的感測器或功能，但這也意味著需要額外的技術支持來進行調整和維護。

> ## Software Setup
> 
> To use the software as it is, upload the Autonomy Software folder to a Raspberry Pi 5, using your preferred SCP method. For beginners we recommend [WinSCP](https://winscp.net/eng/download.php).
> 
> Inside the folder in the raspberry pi create a virtual environment and install the dependencies.
> 
> ```bash
> python3 -m venv venv
> source venv/bin/activate
> pip install -r requirements.txt
> ```
> 
> You must run both the Mavproxy Bridge to interface with the flight controller as well as the main software powering the rest of the robot. For that run in two separate temrinals the scripts:
> 
> ```bash
> start_mavproxy.sh
> ```
> ```bash
> run.sh
> ```
> In the terminal you should see the IP addres to be able to control the robot, if you're connected to the same network just copy paste that on your browser.
> 
> If you would like to be able to control it from different networks and at long distances we recommend you setup [Tailscale](https://tailscale.com/) on your devices.
> 
> For more convenience you can setup these scripts to run automatically on startup, and then use other scripts like `restarter.sh` or `killer.sh` to manage them.
