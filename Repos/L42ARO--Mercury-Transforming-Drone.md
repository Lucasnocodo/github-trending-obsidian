## 專案簡介

Mercury-Transforming-Drone 是一款設計精巧的無人機，具備 1 公斤的內部有效載荷，並採用簡單的變形機制，讓使用者能夠根據需求調整其形狀。這款無人機配備 RGB、深度和熱成像攝影機，並使用 Ardupilot 和 GPS 進行自主導航。使用者需要將自主軟體上傳至 Raspberry Pi 5，並透過命令行啟動 Mavproxy 和主程式，這樣便能在同一網路下控制無人機。這個工具的賣點在於其靈活性和多功能性，適合需要進行多種環境監測的應用場景。

技術上，這個專案使用 Python 進行軟體開發，並依賴於 Raspberry Pi 進行運算，這使得它在硬體需求上相對輕量，但也可能限制了處理能力和擴展性。與其他無人機解決方案相比，例如 AlpinDale/parsync 和 BigBodyCobain/Shadowbroker，Mercury 提供了更靈活的載荷選擇和變形能力，但在處理速度和計算能力上可能不如專業的商業解決方案。實際使用中，使用者可能會面臨網路連接不穩定的問題，建議使用 Tailscale 來解決遠程控制的需求。這個專案目前處於活躍開發階段，適合小型團隊或個人開發者進行實驗和開發，未來有潛力進一步擴展功能和應用場景。

**技術棧**：`Python 3` · `Raspberry Pi 5`

## 重點功能

- 內部有效載荷 1 公斤 — 可根據需求攜帶不同的設備。
- 簡單的變形機制 — 允許無人機根據任務需求調整形狀。
- RGB + 深度 + 熱成像攝影機 — 提供多種監測能力。
- 使用 Ardupilot + GPS — 實現自主導航和控制。
- 支援 Raspberry Pi 5 — 硬體需求相對輕量，易於部署。

## 架構分析

這個專案採用 Raspberry Pi 作為控制核心，因為它提供了足夠的計算能力和靈活性。資料流方面，無人機的攝影機捕捉影像後，透過 Ardupilot 和 GPS 進行導航，並將數據傳輸至控制端。選擇 Raspberry Pi 而非更高效能的硬體，雖然降低了成本，但也限制了處理速度和計算能力。擴展性方面，若要增加更多功能或感測器，可能需要進行硬體升級，這是設計上的一個 trade-off。

## 技術深入分析

Mercury-Transforming-Drone 的核心技術機制在於其變形設計，這使得無人機能夠根據任務需求調整形狀，並搭載不同的感測器。使用 Python 開發的自主軟體能夠在 Raspberry Pi 上運行，這提供了良好的開發靈活性，但也可能在處理速度上有所妥協。無人機的攝影機能夠捕捉 RGB、深度和熱成像影像，並透過 Ardupilot 進行自主導航，這使得它在環境監測方面具備強大的能力。設計上選擇 Raspberry Pi 而非更高效能的硬體，雖然降低了成本，但在處理複雜任務時可能會遇到瓶頸。這個專案的技術風險在於，隨著功能的擴展，可能會對 Raspberry Pi 的性能造成壓力，特別是在需要即時處理大量數據的情況下。整合方面，與主流的無人機控制系統相容性良好，但在 CI/CD pipeline 的整合上可能需要額外的適配工作。

> ## Software Setup
> 
> To use the software as it is, upload the Autonomy Software folder to a Raspberry Pi 5, using your preferred SCP method. For beginners we recommend [WinSCP](https://winscp.net/eng/download.php).
> 
> Inside the folder in the raspberry pi create a virtual environment and install the dependencies.
> 
> ```bash
> python3 -m venv venv
> source venv/bin/activate
> pip install -r requirements.txt
> ```
> 
> You must run both the Mavproxy Bridge to interface with the flight controller as well as the main software powering the rest of the robot. For that run in two separate temrinals the scripts:
> 
> ```bash
> start_mavproxy.sh
> ```
> ```bash
> run.sh
> ```
> In the terminal you should see the IP addres to be able to control the robot, if you're connected to the same network just copy paste that on your browser.
> 
> If you would like to be able to control it from different networks and at long distances we recommend you setup [Tailscale](https://tailscale.com/) on your devices.
> 
> For more convenience you can setup these scripts to run automatically on startup, and then use other scripts like `restarter.sh` or `killer.sh` to manage them.
