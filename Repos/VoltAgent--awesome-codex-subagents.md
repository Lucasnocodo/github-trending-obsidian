## 專案簡介

這個專案是 Codex 子代理的集合，專為特定開發任務設計，幫助開發者在不同的情境下提高效率。使用者可以將這些子代理配置在 Codex 的指定目錄中，透過簡單的指令來啟用，並在需要時明確指示 Codex 使用特定的子代理。這樣的設計使得開發者能夠根據任務需求選擇合適的專家，從而提高工作效率。每個子代理都使用 `.toml` 格式，包含名稱、描述、模型及指令等，並根據任務的複雜度自動選擇合適的模型。這些子代理的設計考量了不同的工作負載，像是安全審核或快速掃描，並提供相應的模型路由。

與其他工具相比，如 0x0funky/agent-sprite-forge 和 0xGF/boneyard，這個專案提供了更為專業化的子代理，能針對具體的開發任務進行優化，並且支持多種語言和框架。實際使用中，開發者可以透過明確的工作流程來協調多個子代理的運作，這在處理複雜的 PR 審核或錯誤調查時特別有用。這個專案的社群活躍度不錯，且有持續的更新，適合需要高效開發流程的團隊使用。對於小型專案或個人開發者來說，可能會覺得這些功能過於複雜。整體而言，這是一個值得關注的工具，尤其是對於需要多樣化開發任務的團隊。

## 重點功能

- 專業化子代理 — 超過 130 種針對不同開發任務的子代理，涵蓋核心開發、語言專家和研究分析等領域。
- 靈活的子代理配置 — 使用者可將子代理配置在全局或專案特定的目錄中，根據需求靈活調整。
- 自動模型路由 — 根據任務複雜度自動選擇合適的 Codex 模型，提升處理效率。
- 沙盒模式控制 — 提供讀取和寫入模式的選擇，確保安全性與靈活性。
- 範例工作流程 — 提供多種實際工作流程範例，幫助用戶快速上手。

> ## Subagent Structure
>

> 
> ```toml
> name = "subagent-name"
> description = "When this agent should be invoked"
> model = "gpt-5.3-codex-spark"
> model_reasoning_effort = "medium"
> sandbox_mode = "read-only"
> 
> [instructions]
> text = """
> You are a [role description and expertise areas]...
> 
> [Agent-specific checklists, patterns, and guidelines]...
> """
> ```
> 
> 
> ### Smart Model Routing
> 
> Each subagent includes a `model` field that automatically routes it to the right model -- balancing quality and cost:
> 
> | Model | When It's Used | Examples |
> |-------|----------------|----------|
> | `gpt-5.4` | Deep reasoning -- architecture reviews, security audits, financial logic | `security-auditor`, `architect-reviewer`, `fintech-engineer` |
> | `gpt-5.3-codex-spark` | Fast scanning, synthesis, and lighter research tasks | `search-specialist`, `docs-researcher`, `agent-installer` |
> 
> 
> ### Sandbox Mode Philosophy
> 
> Each subagent's `sandbox_mode` field controls filesystem access:
> - **Read-only agents** (reviewers, auditors): `sandbox_mode = "read-only"` - analyze without modifying
> - **Workspace-write agents** (developers, engineers): `sandbox_mode = "workspace-write"` - create and modify files
