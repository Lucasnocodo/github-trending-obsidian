## 專案簡介

tg-ws-proxy 是一個本地 MTProto 代理伺服器，旨在加速 Telegram Desktop 的使用體驗。它的運作流程為：Telegram Desktop 透過 MTProto 代理（127.0.0.1:1443）發送請求，代理再將流量轉發至 Telegram 的資料中心，並使用 WebSocket 進行加密傳輸。這樣的設計不僅提高了連接的穩定性，還能在 WebSocket 不可用時自動切換至 TCP 連接，確保用戶的連接不會中斷。tg-ws-proxy 的一句話賣點是「無需第三方伺服器，直接在本地加速 Telegram」。技術上，它使用 Python 開發，並依賴於多個平台特定的庫，如 `pyperclip` 和 `psutil`，這使得它在不同操作系統上均能運行。

與其他類似工具相比，如 AlpinDale/parsync 和 BigBodyCobain/Shadowbroker，tg-ws-proxy 的優勢在於其簡單的安裝過程和本地運行的特性，避免了對外部伺服器的依賴。使用者在安裝和配置上相對簡單，支持 Windows、macOS 和 Linux 系統，並提供 GUI 介面來簡化設置過程。對於需要在不穩定網路環境中使用 Telegram 的用戶，這個工具能有效提升使用體驗。tg-ws-proxy 的社群活躍度中等，開發者在過去的 26 天內持續更新，顯示出對專案的承諾。對於希望在 Telegram 上獲得更流暢體驗的用戶，這是一個值得考慮的選擇。

**技術棧**：`Python 3.8+` · `Docker`

## 重點功能

- MTProto 代理 — 透過本地伺服器加速 Telegram，減少延遲。
- WebSocket 支持 — 使用加密的 WebSocket 連接，確保數據安全。
- 自動切換 — 當 WebSocket 不可用時，自動切換至 TCP 連接。
- 多平台支持 — 支援 Windows、macOS 和 Linux 系統，方便不同用戶使用。
- GUI 配置 — 提供簡單的圖形介面，方便用戶設置和管理代理。

> # TG WS Proxy
> 
> **Локальный MTProto-прокси** для Telegram Desktop, который **ускоряет работу Telegram**, перенаправляя трафик через WebSocket-соединения. Данные передаются в том же зашифрованном виде, а для работы не нужны сторонние сервера.
> 
> ## Как это работает
> 
> ```
> Telegram Desktop → MTProto Proxy (127.0.0.1:1443) → WebSocket → Telegram DC
> ```
> 
> 1. Приложение поднимает MTProto прокси на `127.0.0.1:1443`
> 2. Перехватывает подключения к IP-адресам Telegram
> 3. Извлекает DC ID из MTProto obfuscation init-пакета
> 4. Устанавливает WebSocket (TLS) соединение к соответствующему DC через домены Telegram
> 5. Если WS недоступен (302 redirect) — автоматически переключается на прямое TCP-соединение
>

> | `--log-backups` | `0` | Количество сохранений логов после перезаписи

> ## Конфигурация
> 
> Tray-приложение хранит данные в:
> 
> - **Windows:** `%APPDATA%/TgWsProxy`
> - **macOS:** `~/Library/Application Support/TgWsProxy`
> - **Linux:** `~/.config/TgWsProxy` (или `$XDG_CONFIG_HOME/TgWsProxy`)
> 
> ```json
> {
>   "host": "127.0.0.1",
>   "port": 1443,
>   "secret": "...",
>   "dc_ip": [
>     "2:149.154.167.220",
>     "4:149.154.167.220"
>   ],
>   "verbose": false,
>   "buf_kb": 256,
>   "pool_size": 4,
>   "log_max_mb": 5.0,
>   "check_updates": true
> }
> ```
> 
> Ключ **`check_updates`** — при `true` при запросе к GitHub сравнивается версия с последним релизом (только уведомление и ссылка на страницу загрузки). На Windows в конфиге может быть **`autostart`** (автозапуск при входе в систему).
